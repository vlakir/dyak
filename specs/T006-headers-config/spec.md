# Spec: T006 — Минимизация конфигурации (подстановка по заголовкам колонок)

**Статус:** Analyzed
**Дата создания:** 2026-06-21
**Связанные документы:** `CONCEPT.md` §6, §8–§9 (отклонение фиксируется ADR 2026-06-21), `DECISIONS.md`, `specs/T001-skeleton/spec.md` (переделывает `config.py`/контекст из T001), задел для T002 (склонение + автоопределение пола)

---

## 1. Overview

Убрать обязательный yaml-маппинг и колонку «Пол» из обихода кадровика.
В шаблоне подстановки идут **напрямую по заголовкам колонок таблицы**:
`{{ Фамилия }}`, `{{ Должность }}`, `{{ Дата_начала }}`. Стандартные
кадровые колонки dyak распознаёт по названию автоматически; `dyak.yaml`
становится **опциональным** (только нестандартные имена и `overrides`).
Цель — нулевой порог входа для нетехнического пользователя при сохранении
будущего склонения.

Эта задача — про **подстановку и распознавание колонок** (этап 0-уровня,
без склонения). Само склонение и автоопределение пола реализуются в T002
поверх новой формы контекста.

## 2. User Stories

- Как кадровик, я беру свою обычную таблицу и пишу в шаблоне названия
  колонок как есть — без всякого конфига.
- Как кадровик, я не завожу колонку «Пол»: программа сама определит его
  по ФИО (склонение — в T002).
- Как кадровик с нестандартными заголовками, я при необходимости задаю
  крошечный `dyak.yaml` с алиасами и ручными исправлениями.

## 3. Functional Requirements

- ДОЛЖНА: строить Jinja-контекст из **заголовков колонок** таблицы —
  каждая колонка доступна по своему заголовку.
- ДОЛЖНА: **нормализовать пробелы в заголовках в подчёркивание** для
  доступа в шаблоне: колонка «Дата начала» → тег `{{ Дата_начала }}`.
- ДОЛЖНА: если пользователь **забыл заменить пробел** и написал
  `{{ Дата начала }}` — автоматически чинить (пробел→`_` внутри тегов
  `{{ }}`/`{% %}`) **с предупреждением в лог**. Реализация — на этапе
  патчинга XML в docxtpl (после склейки run'ов), чтобы работало и для
  плейсхолдеров, разрезанных Word'ом.
- ДОЛЖНА: **автораспознавать стандартные кадровые колонки** по заголовку
  (регистронезависимо, с синонимами): фамилия, имя, отчество, должность.
  Они становятся «склоняемыми» (петля для T002); прочие колонки — простая
  строковая подстановка.
- ДОЛЖНА: **поддерживать ФИО в одной колонке** (частая форма таблицы).
  Колонка «ФИО» (синонимы: `ФИО`, `Ф.И.О.`, `Фамилия Имя Отчество`; либо
  `aliases: … fullname`) распознаётся как `fullname` и **построчно
  разбирается** на фамилию/имя/отчество: 3 слова → полное ФИО, 2 слова →
  **без отчества** (пустое отчество — сигнал для автоопределения пола в
  T002), 1 слово → только фамилия, лишние слова приклеиваются к отчеству.
  В контексте доступны и целое `{{ ФИО }}`, и производные `{{ Фамилия }}`/
  `{{ Имя }}`/`{{ Отчество }}` — шаблон единообразен независимо от того,
  одна колонка ФИО в таблице или три. Если в таблице есть и колонка «ФИО»,
  и отдельные колонки фамилии/имени/отчества — приоритет у отдельных,
  «ФИО» доступна только как целый тег (+ warning).
- ДОЛЖНА: работать **без `dyak.yaml`** для типичной таблицы.
- ДОЛЖНА: при наличии `dyak.yaml` принимать опциональные секции: `aliases`
  (нестандартный заголовок → роль surname/name/patronymic/position),
  `overrides` (ручные формы склонения и пола — потребляются в T002).
- ДОЛЖНА: брать шаблон имени выходного файла из флага CLI `--filename`;
  при отсутствии — дефолт `{{ Фамилия }}_{{ Имя }}_{{ Отчество }}.docx`
  (по распознанным колонкам; коллизии — суффикс из T001).
- ДОЛЖНА: **подчищать сдвоенные пробелы** в теле документа после рендера
  (возникают, когда пустая подстановка — напр. отсутствующее отчество —
  оставляет лишний пробел). Чистка идёт по параграфам (вкл. таблицы),
  устойчива к разрезанию по run'ам; 2+ пробела → один.
- НЕ ДОЛЖНА: требовать колонку «Пол».
- НЕ ДОЛЖНА (в этой задаче): склонять, определять пол, синтезировать
  полное «ФИО»/инициалы — это T002. Здесь только плоская подстановка по
  заголовкам + распознавание ролей колонок.

## 4. Success Criteria

- На таблице со стандартными заголовками (без «Пол», без `dyak.yaml`)
  `generate` создаёт документы; теги `{{ Фамилия }}`, `{{ Должность }}`,
  `{{ Дата_начала }}` подставляются.
- Тег с пробелом `{{ Дата начала }}` рендерится корректно + в логе
  warning о нормализации.
- Таблица с одной колонкой «ФИО» генерирует документы: целое `{{ ФИО }}`
  и части `{{ Фамилия }}`/`{{ Имя }}`/`{{ Отчество }}` подставляются;
  строка из 2 слов даёт пустое отчество.
- Нестандартный заголовок, заданный через `aliases`, распознаётся как
  нужная роль.
- 4 проверки зелёные, coverage ≥ 80% на `src/`.

## 5. Key Entities

- **Контекст шаблона** — плоский: ключ = нормализованный заголовок
  колонки, значение = строка ячейки. Поверх — роли (surname/name/
  patronymic/position) для будущего склонения.
- **Распознаватель колонок** — заголовок → роль по словарю синонимов
  (+ `aliases` из конфига). Отдельно распознаёт колонку `fullname` (ФИО
  в одной колонке) и сообщает источник для построчного разбора.
- **Разбор ФИО** (`split_fullname`) — ячейка «ФИО» → (фамилия, имя,
  отчество); 2 слова → пустое отчество. Каноничные ключи контекста
  `Фамилия`/`Имя`/`Отчество`, чтобы шаблон не зависел от формы таблицы.
- **Config (опциональный)** — pydantic-модель: `aliases`, `overrides`.
  Маппинг `columns` и `filename`/`gender` из T001 **упраздняются** в этой
  форме (см. §7, ADR).
- **Нормализатор тегов** — чинит пробелы в `{{ }}` на этапе XML-патча
  docxtpl.

## 6. Assumptions & Constraints

- Jinja-идентификатор не может содержать пробел → отсюда правило
  подчёркивания и автопочинка.
- Заголовки колонок уникальны после нормализации пробелов (коллизия
  «Дата начала»/«Дата_начала» в одной таблице — ошибка с пояснением).
- Пол не хранится в таблице; его автоопределение — ответственность T002.
- Платформа/стек — как в `DECISIONS.md` (Python 3.14, docxtpl и т.д.).

## 7. Out of Scope

- Склонение по падежам, автоопределение пола, фильтр `согл`, синтез
  полного «ФИО» из трёх колонок и инициалов — T002. (Обратный разбор
  одной колонки ФИО на части — **в scope T006**, см. §3.)
- **Сохранение регистра при склонении** (фамилия БОЛЬШИМИ → склонения
  тоже БОЛЬШИМИ) — T002: в T006 подстановка as-is, регистр и так
  сохраняется; проблема возникает только когда движок меняет слово.
- Команда `check`, `init`, PDF — T004/T005.
- CSV/БД-источники.

## Clarify (зафиксировано по итогам обсуждения 2026-06-21)

### Resolved

- **Подстановка по заголовкам, не по yaml-маппингу.** Шаблон ссылается
  на заголовки колонок напрямую; обязательный `columns`-маппинг из T001
  упраздняется.
- **Пробелы в заголовках → подчёркивание** в тегах; забытый пробел dyak
  чинит сам + warning (через XML-патч docxtpl, устойчиво к run-split).
- **Автораспознавание стандартных колонок** (Фамилия/Имя/Отчество/
  Должность + синонимы). yaml опционален: `aliases` для нестандартных
  имён, `overrides` для ручных правок.
- **Имя выходного файла — флаг CLI `--filename`**; дефолт по ФИО, если
  не задан.
- **Колонка «Пол» убирается.** Пол определяется автоматически (в T002):
  правило по отчеству (детерминированно для восточнославянских) →
  fallback pymorphy по имени → не вышло: `override` + сигнал в `check`.
  Проверено пробой: отчество ~100% на стандартных, имя покрывает
  Александр/Олег/Динара, неоднозначные (Саша/Женя) и иностранные — в
  override. `petrovich` сам пол не определяет (поправка к §7.1).

- **ФИО в одной колонке** (дополнение 2026-06-21, по запросу Разработчика).
  Частая форма таблицы — одна колонка «ФИО» вместо трёх. Решено: dyak
  распознаёт её (`fullname`) и разбирает построчно (3/2/1 слово);
  выставляет в контекст и целое `{{ ФИО }}`, и части `{{ Фамилия }}`/
  `{{ Имя }}`/`{{ Отчество }}`. Включено в scope T006 (тот же слой
  распознавания колонок, на котором строится T002). Сохранение регистра
  фамилии при склонении — отнесено к T002 (acceptance в `BACKLOG.md`).

### Open questions

- Полный синтетический тег «ФИО» и инициалы (`{{ ФИО | рд }}`,
  `{{ ФИО.инициалы }}`) — форма уточняется в T002 (нужно ли `ФИО` как
  виртуальная переменная поверх трёх колонок). Прямой `{{ ФИО }}` (целая
  строка без склонения) уже доступен в T006 для одноколоночной формы.

## Analyze

- 🔴 **Critical (миграция).** Переделывает `config.py` и `render/context`
  из T001: модель `Config` меняет смысл (нет обязательного `columns`,
  `gender`, `filename`). Pre-alpha, данных нет — чистый разрыв формата
  допустим; примеры в `examples/` обновить в этом же PR.
- 🟡 **Warning (связка с T002).** Распознавание ролей колонок здесь —
  фундамент для маршрутизации склонения в T002. Контракт «заголовок →
  роль» зафиксировать так, чтобы T002 только навесил движки.
- 🟡 **Warning (автопочинка пробелов).** Полагается на внутренний этап
  патчинга docxtpl; проверить стабильность API (версия 0.20.2) тестом
  на плейсхолдере, разрезанном по run'ам, с пробелом внутри.
- 🟢 **Note.** Дефолтный `--filename` использует распознанные колонки —
  если их нет (только нестандартные заголовки без alias), нужен запасной
  дефолт (порядковый номер) + warning.
- 🟢 **Note.** Коллизия заголовков после нормализации пробелов → явная
  ошибка `TableError` с обоими именами.

## План реализации (TDD, набросок)

1. Распознаватель ролей колонок (синонимы + `aliases`) — чистая логика,
   тесты на словах.
2. Нормализация заголовков → ключи контекста (+ ошибка коллизии).
3. Автопочинка пробелов в тегах на XML-патче docxtpl + warning (тест на
   run-split с пробелом).
4. Упрощённый `Config` (aliases/overrides, всё опционально); `--filename`
   в CLI + дефолт.
5. Обновить `examples/` (убрать «Пол», убрать обязательный yaml,
   шаблон на прямых заголовках) и README/CHANGELOG.
