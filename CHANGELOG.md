# Changelog

История заметных изменений. Формат — упрощённый
[Keep a Changelog](https://keepachangelog.com/).

Записи группируются по версиям или датам релизов. Для проектов без
формального версионирования допустимо использовать дату как заголовок.

Категории:
- **Added** — новая функциональность.
- **Changed** — изменения в существующей функциональности.
- **Fixed** — исправления багов.
- **Removed** — удалённая функциональность.
- **Deprecated** — то, что помечено к удалению, но пока работает.
- **Security** — изменения, важные с точки зрения безопасности.

Если изменение связано с задачей из `BOARD.md` / `BACKLOG.md`,
запись **обязательно** содержит T-ID в скобках, например:
`Added: Превью постов в Telegram (T<NNN>).` Это сохраняет уникальность
T-ID между релизами — `CHANGELOG.md` единственное persistent-
хранилище номеров завершённых задач (см. правило нумерации
в `README.md`).

## [0.1.0] — 2026-06-22

Первый осмысленно завершённый цикл: этапы 0–3 из `CONCEPT.md` —
рабочий пайплайн «таблица + шаблон → кадровые документы» со
склонением ФИО и должностей, автоопределением пола и сухим
прогоном `check`.

### Added
- Надёжность (T004): этап 3 «контроль качества до генерации».
  - Команда `dyak check` (`check.py`): сухой прогон — рендерит каждую
    строку в память (без записи) и печатает отчёт. Виды находок:
    `undefined` (неизвестная переменная — фатально, код 1),
    `not_declined` (склоняемое значение не меняется ни в одном косвенном
    падеже — кандидат на override), `gender_ambiguous` (пол взят
    дефолтом), `gender_mismatch` (ручной `genders` ≠ авто-определение).
  - Фильтр согласования по полу `согл`:
    `{{ ФИО | согл('назначен', 'назначена') }}` → форма по полу ФИО.
  - **Строгий режим Jinja** (`StrictUndefined`): неизвестная переменная
    шаблона → доменная `TemplateError` с именем (а не молчаливая пустая
    подстановка) — и в `generate`, и в `check`, и в имени файла.
  - `resolve_gender` → `GenderResolution { gender, source }` (источник:
    override/отчество/имя/дефолт) для отчёта; `detect_gender` — тонкая
    обёртка для горячего пути.
  - `render_to_document` — общий путь рендера в память (DRY): `generate`
    добавляет запись, `check` только проверяет.
  - Пример `examples/` обновлён: `согл` в шаблоне + раздел про `dyak
    check` в README.
- Склонение должностей (T003): этап 2 «Должности».
  - `inflection/pymorphy_phrase.py`: `PymorphyInflector` — пословное
    склонение фразы через `pymorphy3` (токенизация по пробелам/дефису,
    разбор каждого слова, сборка обратно; несклонившийся токен остаётся
    как есть). Результат кешируется (`lru_cache` по `(text, case)`).
  - `Position` (`Declinable`): должность с ручным `overrides.position`
    приоритетом над движком (полный/частичный override; ключ —
    нормализованный текст должности). Падежный фильтр `{{ Должность | рд }}`
    теперь склоняет (в T002 проходил как есть).
  - `inflection/morph.py`: общий кешированный `MorphAnalyzer` для движка
    пола (T002) и должностей — один экземпляр на процесс (DRY).
  - Пример `examples/` обновлён: должность через падежный фильтр + пример
    `overrides.position` в `dyak.yaml`, вывод перегенерирован.
- Склонение ФИО (T002): этап 1 «ФИО».
  - Падежные фильтры Jinja `им/рд/дт/вн/тв/пр` (`render/filters.py`),
    работают и в теле документа, и в имени файла. Фильтр склоняет любой
    `Declinable`-объект, прочие значения пропускает как есть (поэтому
    `{{ Должность | рд }}` до T003 безопасен).
  - `petrovich`-движок (`inflection/petrovich_fio.py`): склонение
    фамилии/имени/отчества по полу. Обёртка добавляет то, чего нет в
    движке: именительный = исходная форма, пустые части → `''` (не
    `ValueError`), **сохранение регистра** (`ИВАНОВ` → `ИВАНОВА`).
  - Склоняемые доменные объекты (`inflection/fio.py`): `NamePart`,
    `Fio`, `Initials`. В контексте `{{ Фамилия | вн }}`, `{{ ФИО | рд }}`;
    инициалы `{{ ФИО.инициалы }}` / `.инициалы_впереди` / `.инициалы_слитно`
    (корректны и в косвенных падежах — склоняется фамилия).
  - **Автоопределение пола** (`inflection/gender.py`, замена колонки
    «Пол»): отчество (`-вич/-ич/-ыч` → м, `-вна/-чна` → ж) → имя через
    `pymorphy3` → ручной override → дефолт «мужской» + warning.
    `MorphAnalyzer` кешируется и грузится только при необходимости.
  - Опциональная секция `genders` в `dyak.yaml` — ручное указание пола
    для неоднозначных имён (`"Ким Саша": "ж"`).
  - Регрессионная фикстура `tests/fixtures/declension.csv` (м/ж,
    несклоняемые Ким/Шевченко/Дюма, UPPERCASE, двойная фамилия) — 6
    падежей на строку.
  - Пример `examples/` обновлён: шаблон приказа со склонением ФИО и
    инициалами, вывод перегенерирован.
- Минимизация конфига (T006): подстановка **по заголовкам колонок**
  таблицы вместо обязательного yaml-маппинга.
  - `columns.py`: нормализация заголовков (пробел → `_`) и распознавание
    стандартных кадровых колонок (фамилия/имя/отчество/должность +
    синонимы) в «роли» — фундамент маршрутизации склонения для T002.
  - Поддержка **ФИО в одной колонке**: колонка «ФИО» (`ФИО`/`Ф.И.О.`/
    `aliases: fullname`) разбирается построчно на фамилию/имя/отчество
    (3 слова → полное; 2 → без отчества; 1 → только фамилия). В контексте
    доступны и целое `{{ ФИО }}`, и части `{{ Фамилия }}`/`{{ Имя }}`/
    `{{ Отчество }}`; шаблон единообразен независимо от формы таблицы.
  - Автопочинка забытого пробела в тегах `{{ Дата начала }}` →
    `{{ Дата_начала }}` на этапе XML-патча docxtpl (устойчиво к
    разрезанию плейсхолдеров по run'ам) + warning. Трогаются только
    «голые» теги вывода `{{ … }}`; фильтры (`{{ ФИО | рд }}`) и блоки
    `{% … %}` не затрагиваются.
  - CLI-флаг `--filename` (дефолт — по распознанным колонкам ФИО через
    `select|join`, пустые части отбрасываются — без хвостовых `_`;
    запасной порядковый `Документ_N.docx`, если ФИО не распознаны).
  - Подчистка сдвоенных пробелов в теле документа после рендера (когда
    пустая подстановка, напр. отсутствующее отчество, оставляет лишний
    пробел) — по параграфам и таблицам, устойчиво к run-split.
- CI: GitHub Actions workflow `.github/workflows/ci.yml` — прогон
  `ruff check` + `ruff format --check` + `mypy src` + `pytest` на
  Python 3.14 при push в `main` и на каждый PR (ветка `rules/ci`).
- Каркас проекта по методике dreamteam: ТЗ в `CONCEPT.md`, ADR в
  `DECISIONS.md`, задачи T001–T005.
- Этап 0 «Скелет» (T001): команда `generate` — пайплайн «таблица →
  документы» на простой подстановке без склонения.
  - `config.py`: pydantic-схема `dyak.yaml` (маппинг колонок, шаблон
    имени файла; задел под `gender_values`/`overrides`).
  - `io/excel.py`: чтение xlsx, маппинг колонок, валидация обязательных
    полей с номером строки, выбор листа.
  - `render/`: контекст-фасад `сотрудник.*` + рендер docx через
    `docxtpl` (устойчив к разрезанию плейсхолдеров по run'ам) и имени
    файла через Jinja.
  - `io/naming.py`: разрешение коллизий имён (`_2`, `_3`, …).
  - CLI на `typer`, запуск `python -m dyak` (см. ADR 2026-06-21).

### Changed
- T006 переделывает контекст и конфиг из T001: контекст стал плоским
  (ключ = нормализованный заголовок колонки) вместо фасада
  `сотрудник.*`; `dyak.yaml` стал опциональным (секции `aliases` /
  `overrides`). Примеры в `examples/` обновлены под новый формат.

### Removed
- T006: обязательный маппинг `columns`, шаблон `filename` и колонка
  «Пол» (`gender` / `gender_values`) из `dyak.yaml`. Pre-alpha, данных
  нет — чистый разрыв формата без миграции (ADR 2026-06-21).

### Retrospective

- **Что зашло:** ритуал spec/clarify/analyze на крупных фичах (T002,
  T004) ловил слепые зоны до кода; DRY-выносы по ходу (`morph.py` —
  один `MorphAnalyzer` на процесс, `render_to_document` — общий рендер
  в память) держали архитектуру чистой без отдельных рефактор-PR;
  TDD-цикл и зелёные 4 проверки перед каждым push не дали накопить
  долг.
- **Что не зашло:** GitHub по TLS интермиттентно штормил (push/PR/api
  падали с EOF/handshake), приходилось ретраить и поднимать фоновые
  ретраи создания PR — overhead не по вине методики, но замедлял.
- **Правки методики:** введено правило «актуализация `examples/` —
  часть Definition of Done каждой задачи» (PR #5): пример всегда
  отражает текущие возможности dyak и реально прогоняется перед
  закрытием задачи.
