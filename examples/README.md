# Пример запуска dyak

Готовый набор для ручного прогона генерации документов.

| Файл | Назначение |
|---|---|
| `employees.xlsx` | Таблица сотрудников (3 строки), стандартные заголовки, без колонки «Пол». |
| `order_template.docx` | Шаблон приказа со **склонением ФИО и должности** по падежам: `{{ ФИО \| вн }}`, `{{ Должность \| рд }}`, инициалы `{{ ФИО.инициалы_впереди }}`, плюс простые теги по заголовкам (`{{ Дата_начала }}`). |
| `dyak.yaml` | **Опциональный** конфиг (aliases / overrides). Для этого примера не нужен. |

## Старт с нуля: `dyak init` (T005)

Если своего набора ещё нет — `dyak init` положит готовый стартовый
комплект (тот же по смыслу, что файлы рядом), с которого удобно начать:

```bash
PYTHONPATH=src uv run python -m dyak init --dir ./my_project
```

Создаст в каталоге три файла:

- `dyak.yaml` — опциональный конфиг (всё закомментировано);
- `template.docx` — пример приказа со **шпаргалкой падежных фильтров**
  вверху (блок «ШПАРГАЛКА» удаляется перед использованием);
- `table.xlsx` — пример таблицы сотрудников (3 строки).

Сразу после этого набор прогоняется генерацией:

```bash
PYTHONPATH=src uv run python -m dyak generate \
  --table ./my_project/table.xlsx \
  --template ./my_project/template.docx \
  --out ./my_project/output
```

Повторный `init` не затирает существующие файлы — для перезаписи нужен
`--force`.

## Запуск (нулевой конфиг)

Из корня проекта:

```bash
uv sync   # один раз, если ещё не делали

PYTHONPATH=src uv run python -m dyak generate \
  --table examples/employees.xlsx \
  --template examples/order_template.docx \
  --out examples/output \
  --filename "Приказ_{{ Номер_приказа }}_{{ Фамилия }}.docx"
```

Результат — три файла в `examples/output/`:

```
Приказ_17_Иванов.docx
Приказ_18_Петрова.docx
Приказ_19_Ким.docx
```

Внутри — корректно **просклонённые** ФИО и должности (пол определён
автоматически, колонка «Пол» не нужна):

```
Назначить Иванова Петра Семёновича на должность директора ...
Основание: личное заявление Иванова Петра Семёновича.
П. С. Иванов с приказом ознакомлен: ...

Назначить Петрову Анну Сергеевну на должность главного бухгалтера ...
А. С. Петрова с приказом ознакомлена: ...

Назначить Кима Олега Викторовича на должность инженера-программиста ...
```

Обрати внимание: «ознакомлен» для мужчин и «ознакомлена» для женщины —
это фильтр согласования по полу `{{ ФИО | согл('ознакомлен',
'ознакомлена') }}` (T004).

Конфиг (`--config`) не передаётся — подстановка идёт напрямую по
заголовкам колонок таблицы. Имя файла задаётся флагом `--filename`
(тот же Jinja-движок). Если `--filename` опустить, имя соберётся по
распознанным колонкам ФИО: `Иванов_Пётр_Семёнович.docx`.

## Прогресс на больших объёмах (T013)

В интерактивном терминале `generate` сам показывает живой прогресс-бар
(`N/Total` + ETA) — на пакете в тысячи строк видно, что приложение не
зависло. В pipe/файл бар не печатается (не мусорит в лог).

Для интеграции (будущий GUI, T008) есть машиночитаемый поток — флаг
`--progress-json` эмитит построчный JSONL в **stderr**, а stdout
остаётся чистым под результат:

```bash
PYTHONPATH=src uv run python -m dyak generate \
  --table examples/employees.xlsx \
  --template examples/order_template.docx \
  --out examples/output \
  --progress-json 2>progress.jsonl
```

В `progress.jsonl` — события по одному на строку:

```json
{"event": "start", "total": 3}
{"event": "progress", "done": 1, "total": 3, "file": "Иванов_Пётр_Семёнович.docx"}
{"event": "done", "ok": 3, "failed": 0}
```

## PDF-экспорт (T012)

Если нужен ещё и PDF — добавь флаг `--pdf`: рядом с каждым docx ляжет
PDF, сконвертированный через LibreOffice (`soffice --headless`):

```bash
PYTHONPATH=src uv run python -m dyak generate \
  --table examples/employees.xlsx \
  --template examples/order_template.docx \
  --out examples/output \
  --pdf
```

Нужен установленный **LibreOffice** (Linux/macOS/Windows; ищется в PATH и
типичных путях установки). Если его нет — `generate --pdf` завершится
понятной ошибкой и кодом 1 (docx при этом успеют сгенерироваться), а не
трейсбеком.

## Сухой прогон перед генерацией

Перед генерацией сотен файлов прогони проверку — она рендерит всё в
память (ничего не пишет) и печатает отчёт о подозрительных местах
(не просклонившиеся слова, неуверенно определённый пол, опечатки в
тегах шаблона):

```bash
PYTHONPATH=src uv run python -m dyak check \
  --table examples/employees.xlsx \
  --template examples/order_template.docx
```

На этом примере отчёт — «Проблем не найдено» (код возврата 0). Если в
шаблоне опечататься в теге (`{{ Должнсть }}`), `check` назовёт переменную
и завершится кодом 1 — то же самое прервёт и `generate`, поэтому удобно
ловить такое заранее.

## Что попробовать

- **Падежи ФИО.** Поменяй падежный фильтр у `{{ ФИО | … }}`: `им` (кто?),
  `рд` (кого?), `дт` (кому?), `вн` (кого? — винительный), `тв` (кем?),
  `пр` (о ком?). Те же фильтры работают на `{{ Фамилия | … }}`,
  `{{ Имя | … }}`, `{{ Отчество | … }}`.
- **Инициалы.** `{{ ФИО.инициалы }}` → «Иванов П. С.»,
  `{{ ФИО.инициалы_впереди }}` → «П. С. Иванов»,
  `{{ ФИО.инициалы_слитно }}` → «Иванов П.С.» (удобно для имён файлов).
  В косвенных падежах склоняется фамилия: `{{ ФИО.инициалы | рд }}` →
  «Иванова П. С.».
- **Пол определяется сам** по отчеству, а если его нет — по имени.
  Неоднозначные имена (например «Саша» без отчества) можно задать вручную
  в `dyak.yaml`: `genders: { "Ким Саша": "ж" }`.
- **UPPERCASE сохраняется:** если в таблице фамилия записана капсом
  (`ИВАНОВ`), её склонения тоже капсом (`ИВАНОВА`).
- **Должности склоняются** пословно: `{{ Должность | рд }}` → «директора»,
  «главного бухгалтера», «инженера-программиста». Сложную фразу, которую
  движок берёт неидеально (например «заместитель генерального директора»
  в косвенных падежах), можно задать вручную через `overrides.position`
  в `dyak.yaml` (см. закомментированный пример) — ручные формы
  приоритетнее движка.
- В шаблоне теги пишутся **по заголовкам колонок**. Пробел в заголовке
  заменяется на подчёркивание: «Дата начала» → `{{ Дата_начала }}`.
  Если забыть и написать `{{ Дата начала }}` — dyak починит сам и
  напишет warning в лог.
- Добавить строки в `employees.xlsx` — на каждую получится отдельный
  документ.
- Поменять `--filename` — поменяется схема имён выходных файлов.
- Нестандартный заголовок (например «Сотрудник» вместо «Фамилия») —
  привязать к роли через `aliases` в `dyak.yaml` и передать `--config`.
- Заменить три колонки Фамилия/Имя/Отчество одной колонкой «ФИО» (как
  часто бывает в реальных таблицах): dyak сам разберёт её на части.
  В шаблоне будут доступны и целое `{{ ФИО }}`, и `{{ Фамилия }}` /
  `{{ Имя }}` / `{{ Отчество }}`. Если в ячейке два слова — отчество
  будет пустым.

## Важно (текущий этап)

Реализовано склонение **ФИО** (T002) и **должностей** (T003): фамилия/
имя/отчество, целое «ФИО» и должность склоняются по 6 падежам; есть
инициалы в трёх формах; пол определяется автоматически (колонка «Пол» не
нужна); сложные должности закрываются `overrides.position`.

Реализована и **надёжность** (T004): команда `dyak check` (сухой прогон
с отчётом), фильтр согласования по полу `согл`, строгий режим
(неизвестная переменная шаблона → понятная ошибка, а не молчаливая
пустота).

Реализован и **старт с нуля** (T005): команда `dyak init` выкладывает
готовый scaffold-набор (см. раздел «Старт с нуля» выше).

Этап 4 «Опции» завершён: T005 (init), T013 (прогресс + машиночитаемый
JSONL для GUI) и T012 (PDF-экспорт через LibreOffice) — все готовы
(см. разделы «Старт с нуля», «Прогресс на больших объёмах», «PDF-экспорт»).
