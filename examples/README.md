# Пример запуска dyak

Готовый набор для ручного прогона генерации документов.

| Файл | Назначение |
|---|---|
| `employees.xlsx` | Таблица сотрудников (3 строки), стандартные заголовки, без колонки «Пол». |
| `order_template.docx` | Шаблон приказа с тегами прямо по заголовкам колонок: `{{ Фамилия }}`, `{{ Должность }}`, `{{ Дата_начала }}`. |
| `dyak.yaml` | **Опциональный** конфиг (aliases / overrides). Для этого примера не нужен. |

## Запуск (нулевой конфиг)

Из корня проекта:

```bash
uv sync   # один раз, если ещё не делали

PYTHONPATH=src uv run python -m dyak generate \
  --table examples/employees.xlsx \
  --template examples/order_template.docx \
  --out examples/output \
  --filename "Приказ_{{ Номер_приказа }}_{{ Фамилия }}.docx"
```

Результат — три файла в `examples/output/`:

```
Приказ_17_Иванов.docx
Приказ_18_Петрова.docx
Приказ_19_Ким.docx
```

Конфиг (`--config`) не передаётся — подстановка идёт напрямую по
заголовкам колонок таблицы. Имя файла задаётся флагом `--filename`
(тот же Jinja-движок). Если `--filename` опустить, имя соберётся по
распознанным колонкам ФИО: `Иванов_Пётр_Семёнович.docx`.

## Что попробовать

- В шаблоне теги пишутся **по заголовкам колонок**. Пробел в заголовке
  заменяется на подчёркивание: «Дата начала» → `{{ Дата_начала }}`.
  Если забыть и написать `{{ Дата начала }}` — dyak починит сам и
  напишет warning в лог.
- Добавить строки в `employees.xlsx` — на каждую получится отдельный
  документ.
- Поменять `--filename` — поменяется схема имён выходных файлов.
- Нестандартный заголовок (например «Сотрудник» вместо «Фамилия») —
  привязать к роли через `aliases` в `dyak.yaml` и передать `--config`.
- Заменить три колонки Фамилия/Имя/Отчество одной колонкой «ФИО» (как
  часто бывает в реальных таблицах): dyak сам разберёт её на части.
  В шаблоне будут доступны и целое `{{ ФИО }}`, и `{{ Фамилия }}` /
  `{{ Имя }}` / `{{ Отчество }}`. Если в ячейке два слова — отчество
  будет пустым.

## Важно (текущий этап)

Реализована **простая подстановка без склонения** (T006): значения
вставляются ровно как в таблице. Падежные фильтры (`{{ … | вн }}`,
`| рд`), синтез полного «ФИО» и инициалы появятся в T002 («ФИО»).
Колонка «Пол» не нужна — пол будет определяться автоматически в T002.
