# Changelog

История заметных изменений. Формат — упрощённый
[Keep a Changelog](https://keepachangelog.com/).

Записи группируются по версиям или датам релизов. Для проектов без
формального версионирования допустимо использовать дату как заголовок.

Категории:
- **Added** — новая функциональность.
- **Changed** — изменения в существующей функциональности.
- **Fixed** — исправления багов.
- **Removed** — удалённая функциональность.
- **Deprecated** — то, что помечено к удалению, но пока работает.
- **Security** — изменения, важные с точки зрения безопасности.

Если изменение связано с задачей из `BOARD.md` / `BACKLOG.md`,
запись **обязательно** содержит T-ID в скобках, например:
`Added: Превью постов в Telegram (T<NNN>).` Это сохраняет уникальность
T-ID между релизами — `CHANGELOG.md` единственное persistent-
хранилище номеров завершённых задач (см. правило нумерации
в `README.md`).

## [0.3.2] — 2026-06-25

Полировочный цикл по итогам реального теста 0.3.1 кадровиком: спецсимволы в
заголовках (`л/н`), явный именительный `ип`, понятные ошибки шаблона вместо
трейсбек-окон, сохранение намеренных пробелов (T026), и более правильное
склонение фамилий — фамилии-нарицательные (Бивень, Кузнец) теперь остаются в
именительном, как в официальных документах (T027).

### Changed
- **Фамилии-нарицательные не склоняются по умолчанию** (T027) — фамилию
  склоняем, только если pymorphy опознаёт её как фамилию (грамема `Surn`).
  Слова, известные лишь как нарицательные (Бивень, Кузнец, Заяц), остаются в
  **именительном**, как в официальных документах. Причина: petrovich для таких
  слов даёт спорные и порой неверные формы (Кузн**ц**у, Пал**ц**у), а
  именительный всегда безопасен. Обход — список `decline_surnames` в конфиге
  («всё-таки склонять эту фамилию»). `check` показывает оставленные в
  именительном фамилии с подсказкой. ADR в `DECISIONS.md`. Сигнал точный: на
  62 обычных фамилиях — 0 ложных срабатываний.

### Added
- **Явный фильтр именительного падежа `ип`** (T026) — `{{ Фамилия | ип }}`
  даёт именительный, наравне с остальными падежами. Тег без фильтра
  (`{{ Фамилия }}`) по-прежнему тоже даёт именительный (несклоняемые колонки —
  даты/номера — пишутся без фильтра). `им` остаётся тихим алиасом. Справка
  обновлена.

### Fixed
- **Спецсимволы в заголовке колонки ломали шаблон** (T026) — `{{ л/н }}`
  (очень распространённая аббревиатура личного номера) падало с ошибкой
  «`л` is undefined»: Jinja разбирала `/` как деление. Теперь `normalize_header`
  приводит ЛЮБЫЕ спецсимволы (`/`, `№`, `-`, `.` …) к `_` — как и пробелы, — а
  авто-фиксер тегов правит «голые» теги-ссылки (`{{ л/н }}` → `{{ л_н }}`) с
  предупреждением (раз на уникальный тег за прогон, а не на каждый документ);
  теги с фильтром/атрибутом (`| дт`, `.инициалы`) не трогаются.
- **Полный трейсбек во всплывающем окне на грубую ошибку шаблона** (T026) —
  синтаксическая ошибка разметки (`{{ Фамилия | }}` и т.п.) не ловилась
  (`engine.py` ловил только неизвестную переменную) → необработанное исключение
  → PyInstaller показывал окно с трейсбеком. Теперь все ошибки разметки
  оборачиваются в понятное доменное сообщение **в окне лога**; в сборке
  выставлен `disable_windowed_traceback=True` — никаких трейсбек-окон.
- **Удаление намеренных пробелов** (T026) — чистка пробелов (введённая для
  пустых подстановок) бежала по всему абзацу и схлопывала ЛЮБЫЕ кратные пробелы,
  в т.ч. намеренные (выравнивание в подписи). Теперь область чистки — **строго
  подстановки**: трогаются только пробелы вплотную к удалённой пустой
  подстановке; если в абзаце нет пустых подстановок — текст не меняется вовсе.

### Retrospective

- **Что зашло.** Реальный тест кадровиком на боевых данных — лучший источник
  багов: 0.3.0 прошёл все Linux-тесты, но на Windows и на живом приказе вылезли
  6+ проблем, которые синтетика не ловила. Под каждую правку — воспроизведение
  до фикса и регресс-тест после. Для T027 ключевым был **эмпирический поиск
  сигнала** (грамема `Surn`): прогнали 62 фамилии, убедились в 0 ложных
  срабатываний, нашли что petrovich для нарицательных даёт прямо неверные формы
  (Кузнцу/Палцу) — это превратило «непонятно, как правило сформулировать» в
  чёткое правило с обоснованием. `check` сделали прозрачным (показывает, что
  оставлено в им. и почему), чтобы решение было видимым, а не «магией».
- **Что не зашло.** Дважды забыли мелочи при регенерации примера: `--filename`
  (имена файлов уехали в полное ФИО) — поймали по git-renames и переделали.
  По дороге всплыл латентный дефект: `overrides.fio` рекламируется в scaffold,
  но не подключён (→ T028). Стек-PR (#38 поверх #37) потребовал ручного
  rebase `--onto` после merge базового — терпимо, но требует внимательности.
- **Правки методики.** «Definition of Done» для правки, меняющей вывод
  примеров: регенерировать с **теми же флагами**, что в README, и проверять
  `git status` на неожиданные renames. Перед формулировкой нетривиального
  правила (особенно лингвистического) — **измерять сигнал на выборке**, а не
  угадывать; цена пары `uv run python`-прогонов мала, а правило получается
  обоснованным и с понятными границами.

## [0.3.1] — 2026-06-25

Релиз-исправление: v0.3.0 при первом реальном тесте на Windows оказался
непригоден (сломано само склонение). Чинит обе Windows-специфичные проблемы
кодировки (ФИО не склонялось, кракозябры в логе GUI) и **переворачивает
модель склонения** — теперь склоняется любая текстовая колонка, а не только
распознанная по роли. v0.3.0 остаётся помеченным pre-release (сломан).

### Changed
- **Модель «склонение по умолчанию»** (T024) — переворот дефолта: склоняется
  **ЛЮБАЯ** текстовая колонка универсальным фраз-движком (`PhraseInflector`), а
  не только распознанная по роли. Роли усыхают до специализаций: ФИО —
  petrovich+пол; звание — спец-движок; коды/личные номера/даты — литерал (не
  склоняются, по форме значения). Теперь `{{ Подразделение | вн }}` (и любая
  «своя» текстовая колонка) склоняется **без объявления роли**. Многословные
  фразы корректны: голова и согласованные определения склоняются, **родительный
  хвост замораживается** («рота связи» → «роту связи», не «роту связь»),
  учитывается **одушевлённость** винительного (неодуш. «танковый батальон» в
  вин. = им.). **Составные должности** с любыми разделителями: дефис-с-пробелами
  (` - `/` – `/` — `) → две параллельных должности (символ-в-символ); тесный
  дефис → известная лексема целиком («генерал-майору»), иначе каждая часть
  («стрелку-радисту»). **Суперсёдит** band-aid T020 (конкуренция за роль больше
  не нужна — обе колонки склоняются) и **поглощает** T022 (качество фраз).
  Reverse (T007) автоматически разворачивает любую склоняемую колонку в
  падежный тег. ADR в `DECISIONS.md`; новый пример `examples/military_units.xlsx`.

### Fixed
- **Windows: ФИО не склонялось** (T021) — в Windows-сборке v0.3.0 фамилии
  оставались в именительном (ПУПКИН вместо ПУПКИНУ), хотя звание и ядро на
  Linux работали. Корень: `petrovich` открывал свой `rules.json` без явной
  кодировки, поэтому на русской Windows бралась локальная `cp1251` —
  UTF-8-кириллица в суффиксах правил мис-декодировалась, тесты правил не
  совпадали и фамилия молча проходила несклонённой (Linux с UTF-8 по
  умолчанию не затронут). Фикс — обёртка `_Utf8Petrovich`, читающая правила
  явным `encoding='utf-8'`; не зависит от окружения, работает и при прямом
  запуске exe. Добавлен **Windows CI smoke-шаг**: собранный exe генерирует из
  фикстуры, а проверка ассертит склонённую форму в документе (а не только
  факт создания файла — дыра, через которую баг ушёл в релиз, см.
  Retrospective 0.3.0).
- **Windows-GUI: кракозябры в окне лога** (T023) — итоги и warning'и
  отображались битыми символами. Дочерний Python на русской Windows писал
  stdout/stderr в `cp1251` (поток — пайп, не консоль), а GUI декодировал
  UTF-8. Фикс из двух частей: (1) GUI выставляет ядру-подпроцессу
  `PYTHONUTF8=1` через `QProcess.setProcessEnvironment`; (2) ядро по этому
  флагу **явно** переключает `sys.stdout/stderr` на UTF-8 при старте
  (`cli.configure_stdio`) — потому что **frozen-интерпретатор PyInstaller не
  honor-ит `PYTHONUTF8`** для кодировки потоков (выявлено Windows CI: ядро
  падало `UnicodeEncodeError` на выводе кириллицы). Кодировка прямого CLI (без
  `PYTHONUTF8`) не затронута — нативная консоль cp1251. Linux-GUI без изменений.

### Retrospective

- **Что зашло.** Smoke-шаг на **корректность** (а не только факт создания
  файлов) сразу окупился — Windows CI поймал то, что юнит-тесты на Linux в
  принципе не могли: frozen-интерпретатор не honor-ит `PYTHONUTF8`, ядро
  падало `UnicodeEncodeError`. Без этого шага T023-фикс уехал бы «зелёным», но
  неработающим — повтор ошибки 0.3.0. Прототип фраз-движка на 28 кейсах **до**
  продакшн-кода снял риск дёшево: алгоритм заморозки хвоста/одушевлённости
  отлаживался на чистом Python без обвязки. Локальное воспроизведение корня
  каждого бага (cp1251-декод petrovich, омонимия родительного) экономило циклы.
- **Что не зашло.** Windows-фиксы (T021/T023) потребовали **четырёх** прогонов
  Windows CI (~5–10 мин каждый): сначала harness-баг smoke (windowed-exe не
  блокирует шелл), потом два слоя кодировки (PYTHONUTF8 недостаточно во frozen).
  Цена «нет Windows под рукой» — каждый виток через CI. Реальный тест на Windows
  у Владимира вскрыл сразу 4 бага, которые Linux-разработка пропустила — широкий
  слепой зон по платформе.
- **Правки методики.** Для кросс-платформенных фич закладывать **CI-проверку на
  целевой платформе на корректность** в Definition of Done с самого начала (не
  «соберётся — и ладно»). При фиксах кодировки/локали не доверять env-флагам в
  frozen-сборках — проверять явным поведением. «Склонение по умолчанию»
  подтвердило: модель «роль-привратник» была корнем класса багов (T020/T022) —
  generic-дефолт с узкими специализациями надёжнее белого списка ролей.

## [0.3.0] — 2026-06-24

Третий осмысленно завершённый цикл: **графическая оболочка** для
нетехнического кадровика (T008) и **выпуск Windows-бинарников**
(инсталлятор + portable) через GitHub Actions (T010). Проект получил GUI
и первый релизный канал для конечного пользователя без Python; введены
git-теги как релизный триггер (первый тег — `v0.3.0`).

### Added
- **Графическая оболочка** `dyak.gui` (T008) — тонкий фронтенд на PySide6
  над CLI для нетехнического кадровика. Запуск:
  `PYTHONPATH=src uv run python -m dyak.gui`. Базовый набор — две вкладки:
  **Генерация** (выбор таблицы/шаблона/папки и опций, живой прогресс из
  `--progress-json` + кнопка «Отмена») и встроенная **офлайн-справка** по
  разделам. GUI не дублирует логику: генерация запускается через
  `subprocess` (`sys.executable -m dyak generate …`), ядро остаётся
  единственным источником истины (ADR T008). Чистый Qt-free слой `runner`
  (сборка argv, разбор JSONL-прогресса, классификация результата по коду
  возврата) отделён от виджетов; генерация идёт в фоне (`QProcess`), окно
  не «замерзает» на больших пакетах. Новый пример — `examples/gui/`.
- **Иконка приложения** (T008) — `src/dyak/gui/assets/`: вектор `icon.svg`
  (лист + перо — образ приказного писаря-«дьяка»), `icon.png` (256, иконка
  окна в рантайме) и многоразмерный `icon.ico` (16–256) для Windows-сборки
  (T010). Окно/приложение получают иконку через `app.setWindowIcon`.
- Новая runtime-зависимость `PySide6-Essentials` (только для GUI-сценария;
  ядро/CLI её не импортируют). Спайк подтвердил установку и импорт на
  Python 3.14.
- **Сборка Windows-бинарников** GUI «Дьяк» через GitHub Actions (T010):
  по тегу `vX.Y.Z` (или ручным `workflow_dispatch`) на `windows-latest`
  собираются **инсталлятор** `dyak-X.Y.Z-setup.exe` (Inno Setup, per-user
  без админ-прав, RU/EN, ярлык + иконка) и **portable** `dyak-X.Y.Z-
  portable.exe` (единый self-extracting exe, без установки Python и без
  вложенных папок) — два раздельных ассета GitHub Release. Сборка —
  PyInstaller из одного `Analysis` в два таргета (`packaging/dyak.spec`):
  onedir для инсталлятора + onefile для portable, поверх единой точки
  входа `dyak._app_entry`:
  exe без аргументов открывает окно, с аргументами работает как CLI-ядро,
  поэтому GUI вызывает ядро тем же exe через subprocess и в бандле
  (`runner.py` не менялся). Package-data зависимостей склонения
  (`petrovich`, `pymorphy3`, словари) и GUI-ассеты включены в бандл.
  Инструмент сборки — dependency-group `build` (`uv sync --group build`).
  Проект начинает использовать git-теги как релизный триггер (ADR).

### Retrospective

- **Что зашло.** Спайки снова окупились: тяжёлые риски снимали до кода и
  дёшево — PySide6 на Python 3.14 (T008), subprocess-ядро в
  PyInstaller-бандле и onefile self-relaunch (T010). «Тонкий фронтенд»
  T008 окупился в T010 — `runner.py` не пришлось менять, бандл заработал
  как есть. Единая точка входа `_app_entry` (GUI без аргументов / CLI с
  аргументами) дала один exe для обоих режимов без условных импортов.
- **Что не зашло.** GitHub TLS-штормил почти всю серию — merge/push/PR
  через REST с ретраями, заметный overhead. По T010 потребовалось два
  круга правок по **формату поставки** артефактов (потерянный setup.exe;
  затем portable zip→единый exe и раздельные артефакты) — формат выкладки
  не был проговорён заранее. Плюс «zip от GitHub» у тестовых артефактов
  ввёл в заблуждение (сырые exe — только в Release по тегу).
- **Правки методики.** Для инфра-задач со сборкой/поставкой — фиксировать
  **формат артефактов** (один файл vs папка, raw vs архив, отдельные vs
  общий) в plan/spec **до** написания workflow; добавить это в
  «definition of done» build-задач. workflow_dispatch запускается только
  из default-ветки — закладывать «проверка сборки только после merge» в
  план инфра-фич, а не ожидать прогон на PR.

## [0.2.0] — 2026-06-23

Второй осмысленно завершённый цикл: этап 4 «Опции» (`init` / PDF /
прогресс), обратная сборка шаблона `reverse` и большой блок поддержки
госструктур — распознавание ролей колонок по содержимому, склонение
воинских/служебных званий (в т.ч. в `reverse`), чистка пустых ячеек,
учёт фильтра/скрытых строк и предупреждение о подписи. Плюс фикс
экранирования имени файла. Спайк T015 (нейро-теггер для омонимии
падежа) проведён и тема осознанно отложена (ADR), без новых тяжёлых
зависимостей в релизе.

### Added
- Учёт фильтра/скрытых строк xlsx (T019): при пакетной генерации документы
  формируются **только из видимых** строк (не скрытых автофильтром или
  вручную), а контентное распознавание ролей (T016) использует образцы со
  **всех** строк, включая скрытые — фильтр управляет выборкой документов, не
  обедняя распознавание. `io/excel.py` грузит книгу без `read_only` (иначе
  openpyxl не отдаёт состояние скрытых строк), проверяет
  `ws.row_dimensions[r].hidden`, логирует число пропущенных скрытых строк.
  Единый эффект для `generate`/`check`/`reverse`. ADR 2026-06-23 (отказ от
  `read_only`-оптимизации; экстремальные объёмы — T009).
- Предупреждение reverse о подписи/исполнителе (T018): новая находка отчёта
  `signature_risk` (раздел «Проверьте подпись/исполнителя») — когда имя
  субъекта заменено тегами в нескольких местах (≥2), reverse предупреждает,
  что одно из них может быть подписью или блоком исполнителя (там фамилия
  обычно постоянная, тег лишний). Счёт по всей семье ФИО-кандидатов (целое
  ФИО + части + инициалы, флаг `Candidate.name_part`), т.к. тело и
  подпись-инициалами дают совпадения на разных кандидатов. «Слепого» режима у
  reverse нет — чужая фамилия (другой подписант/исполнитель) не тегируется;
  предупреждение закрывает лишь случай совпадения фамилии субъекта и
  подписанта. ADR 2026-06-23.
- Склонение званий в `reverse` (T017): `dyak reverse` разворачивает колонку
  «Звание» в падежный тег (`{{ Звание | дт }}`) через тот же decline-and-match,
  что ФИО/должности. `Rank` добавлен в isinstance склоняемых кандидатов
  (`reverse/candidates.py`); движок `rank_inflector`/`rank_overrides` проброшен
  через `reverse_template` → `build_template` → `build_candidates` и
  `_verify_roundtrip` (round-trip воспроизводит тег, ложного `roundtrip_mismatch`
  нет). Составное звание: голова склоняется, генитивный хвост в теге сохраняется;
  омонимия головы → `ambiguous` (как у ФИО/должностей); `overrides.rank`
  подхватывается; «Личный номер» (несклоняемый) матчится точным совпадением.
  Закрывает хвост T016 фазы B (reverse не склонял звания). Пример reverse со
  званием в `examples/README`. ADR 2026-06-22.
- Чистка пустых ячеек (T016, фаза C): пустая опциональная подстановка не
  оставляет лишнего пробела или висячей пунктуации. **Empty-aware** механизм:
  Jinja `finalize`-хук помечает пустой результат `{{ … }}` служебным маркером
  (`EMPTY_MARKER`, символ Private Use Area), а пост-обработка параграфа
  (`render/engine.py`, `_clean_paragraph`, маска по склейке run'ов) снимает
  маркер и при пустом значении убирает висячую завершающую пунктуацию
  (`. ! ? …`) в конце параграфа, сохраняя разделители (`, ;`) и лейбл:
  «Звание: ▮.» → «Звание:», «Оклад ▮, надбавка.» → «Оклад, надбавка.»,
  ведущий пробел от пустого первого значения убирается. Плюс общая чистка —
  схлопывание кратных пробелов, пробел перед закрывающей пунктуацией, края
  параграфа. Маркер транзитный (снимается до сохранения/сверки) — reverse
  round-trip и `check` не ломаются; в имени файла снимается отдельно.
  Заменяет `_collapse_paragraph_spaces`. ADR 2026-06-22.
- Склонение воинских/служебных **званий** (T016, фаза B): движок
  `RankInflector` (`inflection/rank.py`) — гибрид «голова склоняется +
  генитивный хвост замирает». Согласованное прилагательное склоняется с
  головой («старший лейтенант» → «старшему лейтенанту»); хвост от маркера
  (числовой токен «3 ранга», имя собственное «Российской Федерации»,
  курируемые маркер-существительные службы/юстиции/полиции/ранга/класса/…
  с откатом через прилагательное «медицинской службы») замирает в
  родительном; дефис склоняет только последний сегмент («контр-адмиралу»,
  «генерал-майору»). Чинит спайк-поломки пословного движка («майору
  медицинской службе», «капитану 3 рангу», «майором полицией», «контрам-
  адмиралу», потерю регистра имён собственных). Новый `Declinable`-объект
  `Rank` + секция `overrides.rank` в `dyak.yaml` (ручной escape-hatch,
  зеркало `overrides.position`); роль `rank` маршрутизируется в `generate`
  и `check` (там же эвристика `not_declined` теперь покрывает звания).
  Общий примитив `inflect_word` вынесен из движка фраз (DRY). Reverse (T007)
  звание пока не склоняет — параметры `rank_*` ему не переданы (не регрессия,
  запарковано). Пример `examples/`: `personnel_ranks.xlsx` +
  `rank_order_template.docx` (благодарность с дательным звания). ADR 2026-06-22.
- Распознавание ролей колонок **по содержимому** (T016, фаза A): когда
  заголовок незнаком словарю синонимов, роль определяется по образцам ячеек
  (`columns.infer_role` на уже имеющемся `pymorphy3`, 0 новых зависимостей).
  Грамемы `Surn`/`Name`/`Patr` (по максимальному score разбора) дают
  фамилию/имя/отчество и целое ФИО; словарь вершинных маркеров званий —
  роль `rank`; маска кода — `personal_number`; маска даты исключает колонку
  из текстовых ролей; остаток текста → `position`. Нестандартные заголовки
  («Сотрудник», «Наименование должности») работают **без** `aliases`.
  Приоритет: `aliases` > заголовок-синоним > содержимое, но единогласный
  контент перебивает заголовок при **жёстком пороге** (≥3 образца, все
  согласны, заголовок не подтверждён ни одной ячейкой — ловит ошибку
  разметки), иначе заголовок главнее; неуверенное распознавание → лучшая
  догадка + warning. Без образцов `recognize()` работает как в T006
  (header-only, pymorphy не зовётся — нулевая регрессия). Новые роли
  `rank`/`personal_number` в `Role`; в фазе A только распознаются (склонение
  званий — фаза B, личный номер несклоняем). Пример `examples/`:
  `employees_nonstandard.xlsx` (колонка «Сотрудник» без aliases) + раздел
  README. ADR 2026-06-22.
- Обратная сборка шаблона (T007): команда `dyak reverse --doc … --table …
  --row N --out …` собирает docx-шаблон из готового заполненного документа
  и строки таблицы, по которой он сделан. Deterministic-стек на уже
  имеющихся petrovich + pymorphy3 (0 новых зависимостей): **decline-and-
  match** ставит падежные теги (`{{ Фамилия | дт }}`), сверяя формы по 6
  падежам; **run-rewriting** вставляет тег целиком в один run (устойчиво к
  дроблению Word по run'ам); омонимия падежа → первый по порядку + пометка
  `ambiguous` + жёлтая подсветка спорного места в шаблоне; **round-trip
  verify** прогоняет собранный шаблон обратно через `generate` и сверяет с
  образцом (`roundtrip_mismatch`). Best-effort: отчёт честно делит находки
  на `replaced` / `ambiguous` / `not_found` / `unmatched_text`. Нейро-
  теггер slovnet (омонимия по контексту) и локальный LLM-fallback осознанно
  отложены в T015 / T014. ADR 2026-06-22.
- PDF-экспорт (T012): флаг `generate --pdf` конвертирует готовые docx в
  PDF через LibreOffice headless (`soffice --convert-to pdf`, один
  батч-вызов рядом с docx). Кросс-платформенный поиск `soffice`
  (PATH + типичные пути Linux/macOS/Windows); LibreOffice не найден →
  понятная ошибка и код 1, без трейсбека. Прямой вызов `soffice` (не
  `docx2pdf`, тот требует MS Word). ADR 2026-06-22.
- Прогресс `generate` (T013): живой `rich`-бар в интерактивном терминале
  (`N/Total`); в pipe/файл бар не печатается. Флаг `--progress-json`
  эмитит построчный JSONL в stderr (события `start`/`progress`/`done`/
  `error`) — контракт CLI↔GUI для будущего GUI (T008); stdout остаётся
  чист под результат. Новая зависимость `rich`. ADR 2026-06-22.
- Команда `dyak init` (T005): выкладывает стартовый scaffold-набор —
  `dyak.yaml`, `template.docx` со шпаргалкой падежных фильтров и
  `table.xlsx`; `--dir` задаёт каталог, `--force` перезаписывает.
  Кадровик стартует с рабочего примера, а не с пустого листа. Ассеты —
  bundled package-data (`dyak/scaffold/`, ADR 2026-06-22). Этап 4 «Опции»
  разбит на T005 (init) / T012 (PDF) / T013 (прогресс).

### Fixed
- HTML-спецсимволы в имени файла (T011): редкий `&`/`'`/`<`/`>`/`"` в
  значении (ФИО, орг.) больше не уезжает в `&amp;`/`&#39;`/… в **имени**
  выходного файла. Общий с телом docx Jinja-env держит `autoescape=True`
  (обязателен для валидного XML тела), поэтому `render_filename` снимает
  HTML-экранирование обратным `html.unescape`. Тело docx по-прежнему
  экранируется корректно (свой путь рендера не затронут).

### Retrospective

- **Что зашло:** зонтичная задача в фазы под одним T-ID (T016 A/B/C —
  распознавание / склонение званий / чистка ячеек) дала связный крупный
  блок без дробления на искусственные мелкие задачи, при этом каждая фаза
  — отдельный PR; «спайк-first» для фичи с тяжёлой зависимостью (T015)
  окупился сразу — измерил цену (slovnet ~75% на целевой нише) до того,
  как тащить numpy + ~29 МБ моделей, и решение отложить принято на цифрах,
  а не на ощущениях; decline-and-match переиспользован forward↔reverse и
  расширен на звания (T017) без дублирования логики (DRY держал
  архитектуру); для мелких производных вопросов Владимира (T017/T018/T019)
  «spec-lite» (без полного ритуала, но с acceptance + ADR) оказался
  правильным масштабом — полный spec/clarify/analyze был бы overhead.
- **Что не зашло:** сторонний ревью де-факто выпал на всю серию —
  CodeRabbit упёрся в rate-limit (кончились кредиты организации), qodo на
  паузе по подписке; полагались на self-review + зелёный CI (по
  договорённости с Владимиром мердж после зелёного CI, когда бот
  недоступен). GitHub по TLS продолжал интермиттентно штормить
  (push/merge/api с EOF/handshake) — merge стабильно шёл через REST
  `gh api`, обычный `gh pr merge` отдавал битый JSON; overhead не по вине
  методики, но устойчиво раздражал всю серию.
- **Правки методики:** закрепить «спайк-first» как явный шаг для любой
  фичи с тяжёлой/неочевидной зависимостью (подтверждать рантайм и мерить
  выигрыш до интеграции) — в этом релизе спас от напрасной зависимости;
  при срезе milestone не забывать чистить `BOARD.md → Done` (на 0.1.0
  забыли — задачи 0.1.0 провисели в Done до 0.2.0; теперь Done чистится
  как часть ритуала релиза).
