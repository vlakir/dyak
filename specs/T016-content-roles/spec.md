## 1. Overview

Сейчас dyak определяет роль колонки (фамилия / имя / отчество / ФИО /
должность) **только по названию заголовка** через статический словарь
синонимов (`'фио'`, `'должность'`, …) плюс ручные `aliases` в `dyak.yaml`.
Любой нестандартный заголовок — «Фамилия, имя, отчество», «Сотрудник»,
«Наименование должности» — не распознаётся, и кадровик вынужден лезть в
yaml. Цель T016 — научить dyak узнавать роль **по содержимому ячеек**
(на уже имеющемся `pymorphy3`, без новых зависимостей), оставив заголовок
быстрым подтверждением, а `aliases` — ручным override с высшим приоритетом.

Попутно (всё всплыло вокруг кадровых таблиц **госструктур** — армия, флот,
МВД, военно-медицинская и гражданская служба) задача закрывает две смежные
боли: корректное **склонение составных воинских/служебных званий** и
**чистый пропуск на месте пустой опциональной ячейки** (звание, отчество)
без лишних пробелов и артефактов.

## 2. User Stories

- **Как кадровик**, я хочу, чтобы dyak понимал мою таблицу с заголовками
  «Фамилия, имя, отчество», «Сотрудник», «Наименование должности» **без
  правки `dyak.yaml`**, чтобы не разбираться в конфиге.
- **Как кадровик госструктуры**, я хочу колонки «Звание» и «Личный номер»,
  чтобы звание подставлялось в нужном падеже («майору медицинской службы»),
  а личный номер — буквально.
- **Как кадровик**, я хочу, чтобы при пустой ячейке (нет отчества/звания) в
  готовом документе был **чистый пропуск**, а не «двойной пробел», «пробел
  перед точкой» или ошибка.
- **Как кадровик**, я хочу, чтобы при неуверенном распознавании dyak
  **предупредил** (в `check`/отчёте), а не молча подставил не то.

## ФАЗА A — Распознавание ролей по содержимому (`columns`) — ЯДРО

### 3A. Functional Requirements

- **ДОЛЖНА** определять роль колонки по образцам её ячеек (3–5 непустых
  строк), когда заголовок незнаком словарю синонимов:
  - ячейка из 2–3 слов с грамемами `Surn`+`Name`(+`Patr`) → роль `fullname`
    (одна колонка ФИО, разбирается как сейчас);
  - отдельные колонки с доминирующей грамемой `Surn` / `Name` / `Patr` →
    `surname` / `name` / `patronymic`;
  - колонка, парсящаяся как дата → служебно помечается датой (не роль
    склонения, но защищает от ложного зачисления в текстовые роли);
  - остаток текстовых колонок без name-грамем → кандидат на `position`.
- **ДОЛЖНА** соблюдать приоритет источников роли (уточнено в Clarify A1):
  `aliases` (ручной) — абсолютный верх; ниже — заголовок-синоним и
  контент, причём **уверенный контент может перебить заголовок-синоним**
  при явном расхождении (ловит ошибку разметки), пометив расхождение
  предупреждением. Порог «уверенности» для перебивания — см. Analyze 🔴-1.
- **ДОЛЖНА** распознавать новые роли-колонки **«Звание»** и **«Личный
  номер»**. «Звание» — по заголовку-синониму/aliases **и по содержимому
  через встроенный словарь маркеров званий** (рядовой…маршал, флотские,
  служебные; Clarify A4). «Личный номер» — несклоняемая роль (подстановка
  буквальная, как обычная ячейка; распознавание — заголовок/aliases, по
  содержимому опционально: маска «цифро-буквенный код»).
- **ДОЛЖНА** при неуверенности (несколько колонок претендуют на роль; ни
  заголовок, ни контент не дали уверенного ответа) присваивать **лучшую
  догадку** и сопровождать её **предупреждением** (`logger.warning` +
  строка в отчёте `check`), а не делать тихий выбор и не падать (Clarify A2).
- **ДОЛЖНА** вести **встроенный словарь маркеров званий** как
  дополнительный контент-сигнал для роли «Звание» (Clarify A4).
- **НЕ ДОЛЖНА** менять поведение на таблицах, которые уже распознаются по
  заголовку (нулевая регрессия T001–T007): контент-эвристика — только
  fallback, когда заголовок неизвестен.
- **НЕ ДОЛЖНА** добавлять новые runtime-зависимости (только `pymorphy3`,
  уже в стеке).

### 4A. Success Criteria

- Таблица с заголовками «Фамилия, имя, отчество», «Сотрудник»,
  «Наименование должности» распознаётся корректно **без** `aliases`.
- `aliases` перекрывает контентную догадку (тест на конфликт).
- Неоднозначная колонка → предупреждение/отчёт (не тихая ошибка).
- Несклоняемые/женские фамилии (Шевченко, Дюма, Ким-ж) не ломают
  контент-распознавание (ниже score — но роль определяется верно).
- Колонки «Звание»/«Личный номер» опознаются; личный номер подставляется
  буквально.
- Существующие таблицы T001–T007 не регрессируют.
- 4 проверки зелёные, coverage ≥ 80% на `src/` (целевой ~93%).

### 7A. Out of Scope (фаза A)

- Качество склонения званий — фаза B.
- Чистка пробелов — фаза C.
- LLM-распознавание ролей — нет (детерминированная эвристика).

## ФАЗА B — Склонение составных званий (`inflection`)

### 3B. Functional Requirements

- **ДОЛЖНА** склонять **простые однословные** звания не хуже текущего
  движка должностей (рядовой / сержант / лейтенант / капитан / майор /
  полковник / мичман — регрессия не допускается).
- **ДОЛЖНА** для **составных** званий склонять только **вершинное**
  существительное, оставляя несогласованный хвост в родительном:
  - «майор медицинской службы» → дт «майору медицинской **службы**»
    (не «…службе»);
  - «капитан 3 ранга» → дт «капитану 3 **ранга**» (не «…рангу»);
  - «советник юстиции 1 класса» → дт «советнику юстиции 1 **класса**».
- **ДОЛЖНА** сохранять регистр имён собственных в хвосте («маршал
  Российской Федерации» — не «российской федерации»).
- **ДОЛЖНА** корректно обрабатывать дефисные служебные части званий
  («контр-адмирал», «вице-адмирал») — не склонять служебную приставку.
- **МОЖЕТ** опираться на `overrides.position`-механизм (или новый
  `overrides.rank`) как явный escape-hatch (решить в Clarify B1/B2).
- **НЕ ДОЛЖНА** регрессировать склонение должностей (T003).

### 4B. Success Criteria

- Прогон по 6 падежам: простые звания — как раньше; составные
  (медслужба / флот «N ранга» / госслужба / дефисные) дают корректный
  падеж вершины, хвост в родительном, регистр имён собственных сохранён —
  хотя бы через явный механизм.
- Несогласованный хвост заморожен в родительном во всех косвенных падежах.
- 4 проверки зелёные, coverage в норме.

### 7B. Out of Scope (фаза B)

- Полный морфологический парсер синтаксических зависимостей — нет
  (эвристика «вершина + маркеры хвоста» или словарь).
- Распознавание колонки «Звание» по содержимому — это фаза A.

### Spike-находки по склонению званий (2026-06-22, перенесено из BACKLOG)

Прогон текущего движка должностей (`PymorphyInflector`, пословно) по 6
падежам:
- ✅ **простые однословные** склоняются верно: рядовой/сержант/прапорщик/
  лейтенант/капитан/майор/подполковник/полковник; флотский мичман.
- ❌ **составные ломаются** — пословный движок тащит в целевой падеж
  **несогласованное определение** (родительный хвост), который должен
  замереть в родительном: военно-медицинские «майору медицинской **службе**»
  (надо «…службы»), флотские «капитану 3 **рангу**» (надо «…ранга»),
  гражданские «советнику юстиции 1 **классу**» (надо «…класса»); дефисные
  служебные части склоняются как слова («**контрам**-адмиралу», «**вицей**-
  адмиралом»); теряется регистр имён собственных («маршалу **российской
  федерации**»).
- ⚠️ полицейские в дательном «майору полиции» верны лишь по **совпадению**
  форм слова «полиция» (рд=дт=пр) — в творительном уже ломаются («майором
  полиц**ией**»), т.е. «работает» не считается.

**Корень:** склонять только **вершинное** существительное, хвост (после
маркеров «медицинской службы»/«полиции»/«N ранга»/«юстиции»/«N класса»)
оставлять в родительном. Обходной путь `overrides.position` (ручные
`CaseForms` по фразе) **работает уже сейчас**.

## ФАЗА C — Чистка пробелов у пустых ячеек (`render`)

### 3C. Functional Requirements

- **ДОЛЖНА** при пустой подстановке убирать **один** лишний пробел:
  - перед закрывающей пунктуацией (`. , ; : ! ? » ) …`);
  - в начале и конце параграфа.
- **ДОЛЖНА** работать через границы run'ов (как `_collapse_paragraph_spaces`
  в T006), не ломая форматирование.
- **НЕ ДОЛЖНА** трогать тире, открывающие кавычки/скобки, неразрывные
  пробелы.
- **НЕ ДОЛЖНА** регрессировать непустые подстановки и таблицы T001–T007.

### 4C. Success Criteria

- `Звание: {{ Звание }}.` при пустом звании → «Звание: .» **без** пробела
  перед точкой → «Звание:.» (или решить целевой вид в Clarify C1).
- Пустое первое значение не оставляет ведущего пробела в параграфе.
- Тире/кавычки/скобки не затронуты (тест).
- 4 проверки зелёные.

### 7C. Out of Scope (фаза C)

- Полное **исчезновение** строки/блока при пустом поле (`{% if %}`) —
  отдельная история, по-крупному, не здесь.

### Spike-находки по пустым ячейкам (2026-06-22, перенесено из BACKLOG)

Рендер с пустой опциональной ячейкой (звание/отчество):
- ✅ **ошибки нет** — пустая ячейка это определённая переменная `''`,
  `StrictUndefined` её не трогает; падежный фильтр на `''` тоже не падает;
  остатка тега нет.
- ✅ **середина фразы чистая** — `Назначить {{ Звание }} {{ Фамилия }}.`
  при пустом звании → «Назначить Петров.» (сдвоенный пробел схлопывает
  `_collapse_paragraph_spaces`, T006).
- ⚠️ **остаются 3 косметических артефакта по одному пробелу:** пробел перед
  закрывающей пунктуацией (`Звание: .`, `Оклад , надбавка.`) и пробел в
  начале строки, если пустое значение стоит первым (`·Петров назначен.`).

**Корень:** текущая чистка (`_collapse_paragraph_spaces`) лишь схлопывает 2+
пробела в один; не убирает пробел перед закрывающей пунктуацией и не
подрезает края параграфа. Делать аккуратно через границы run'ов, не трогая
тире/открывающие кавычки/скобки.

## 5. Key Entities

- **`Recognition`** (`columns.py`) — результат распознавания: `roles`
  (роль → ключ контекста), `fullname_source`. T016 расширяет источники
  определения роли (контент) и набор ролей (`rank`, `personal_number`?).
- **`Role`** (`config.py`) — `Literal[...]`; добавляются роли звания и
  личного номера (имена — в Clarify A3).
- **Контентная эвристика** — функция «по образцам ячеек → роль», на
  грамемах `pymorphy3` (`Surn`/`Name`/`Patr`), скоринг по нескольким
  строкам.
- **Движок склонения званий** (фаза B) — расширение/обёртка над
  `PymorphyInflector` либо словарь + `overrides`.

## 6. Assumptions & Constraints

- Пользователь — нетехнический кадровик; цель — минимум правки конфига.
- 0 новых runtime-зависимостей (только `pymorphy3`/`petrovich` из стека).
- Образцы для контент-эвристики — первые N непустых ячеек колонки
  (таблицы кадровые, однородные по колонке).
- Python 3.14, `uv`, ruff(ALL)+mypy 0 ошибок, pytest coverage ≥ 80%.
- `aliases`/`overrides` остаются ручным override с высшим приоритетом.

## 7. Out of Scope (общий)

- Распознавание произвольных не-кадровых таблиц (фокус — кадровые роли).
- Условное исчезновение блоков при пустом поле (`{% if %}`).
- См. также пофазные Out of Scope (7A/7B/7C).

### Resolved (с ответами Разработчика 2026-06-22)

- **A1 (приоритет):** **контент может перебить заголовок-синоним** при

- **A2 (неоднозначность):** **лучшая догадка + warning** (не «без роли»,
  не жёсткая ошибка). Неуверенная колонка получает наиболее вероятную
  роль, dyak предупреждает (`logger.warning` + строка отчёта `check`).
- **A4 (распознавание «Звание»):** **словарь маркеров званий** — заголовок-
  синоним/aliases + встроенный список званий как контент-сигнал.

### 🔴 Critical (фиксим до кода)

> **🔴-1 РАЗРЕШЕНО (Разработчик 2026-06-22):** принят **жёсткий порог** —
> контент перебивает заголовок-синоним ТОЛЬКО когда (а) **все** образцы
> (≥3 из 5) согласны на другую роль И (б) роль заголовка **не
> подтверждается ни в одной** ячейке. Иначе заголовок остаётся, расхождение
> логируется. Нулевая регрессия T001–T007 защищена.

- **🔴-1. «Контент перебивает заголовок» без порога — главный риск
  регрессии.** Ответ A1 разворачивает ранее согласованный гибрид
  («заголовок главнее, контент заполняет пробелы») в «контент может
  перебить». Без жёсткого порога контентная эвристика может **ошибочно
  переопределить ВЕРНЫЙ заголовок** и сломать таблицы T001–T007 (требование
  нулевой регрессии 4A). Нужно определить, **когда** контент достаточно
  уверен, чтобы перебить явный заголовок-синоним. Предлагаемый порог:
  контент перебивает заголовок ТОЛЬКО если (а) **все** образцы (3–5/5)
  согласны на другую роль И (б) роль из заголовка образцами **не
  подтверждается ни в одной** ячейке (напр. заголовок «Имя», но ни в одной
  ячейке нет грамемы `Name`, а во всех — `Surn`). Иначе — заголовок
  остаётся, расхождение лишь логируется. → **нужно подтверждение порога
  Разработчиком.**

### 🟡 Warning (обсудить)

- **🟡-1. Что считать «образцом» и порог согласия.** Нужно зафиксировать:
  N = первые 5 непустых ячеек; роль присваивается по содержимому, если
  ≥ 60% (3/5) образцов дают её доминирующей грамемой. Пустые/однострочные
  таблицы (1–2 строки данных) — деградация к заголовку-only (мало образцов
  → не перебиваем заголовок). Зафиксировать в ADR.
- **🟡-2. Грамемная неоднозначность Surn vs Name vs Position.** Многие
  слова имеют несколько разборов pymorphy (фамилия «Кузнецов» = Surn, но
  «кузнец» — обычное сущ.; имя «Вера» = Name и сущ. «вера»). Нужен скоринг
  по **максимальному score разбора** с нужной грамемой, а не «есть ли
  грамема вообще», иначе должности с фамилие-подобными словами уедут в
  surname. Зафиксировать алгоритм скоринга в ADR.
- **🟡-3. Словарь маркеров званий — объём и хранение.** Где живёт

  ефрейтор/сержанты/прапорщики/офицеры/высший комсостав + флот + спец-
  звания МВД/юстиции). Риск: неполный словарь молча не опознаёт звание.
  Решить в ADR; вероятно — компактный набор **вершинных** слов званий
  (майор/капитан/полковник/мичман/…), а не всех словосочетаний.
- **🟡-4. Конфликт с разбором одной колонки «ФИО».** Текущая логика: если
  есть `fullname` И отдельные surname/name/patronymic — `fullname` не
  разбирается. Контент-эвристика может теперь **создать** роль surname из
  безымянной колонки и непреднамеренно отключить разбор ФИО. Проверить

  как заголовочные. Тест на таблицу «ФИО (одна колонка) + контентно
  опознанная фамилия в другой».
