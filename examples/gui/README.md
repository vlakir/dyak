# Пример: графическая оболочка dyak (GUI, T008)

Графическое окно для тех, кто не работает с терминалом. Это **то же
ядро dyak**, что и в командной строке, — окно лишь собирает за вас
команду и показывает прогресс и результат (тонкий фронтенд над CLI,
ADR T008). Всё работает **офлайн**.

## Запуск

Из корня проекта:

```bash
uv sync                 # один раз, если ещё не делали
PYTHONPATH=src uv run python -m dyak.gui
```

(`PYTHONPATH=src` нужен потому, что проект сознательно без `[build-system]`
— пакет не устанавливается в окружение, запускается из исходников, как и
CLI: `PYTHONPATH=src uv run python -m dyak …`. Окно пробрасывает это
окружение в ядро, которое зовёт через `subprocess`.)

Откроется окно с двумя вкладками — базовый набор для нетехнического
пользователя:

| Вкладка | Что делает | Команда ядра |
|---|---|---|
| **Генерация** | Пакетная генерация документов с живым прогрессом и кнопкой «Отмена». | `dyak generate` |
| **Справка** | Встроенный офлайн-help по разделам. | — |

Остальные команды (`check` / `init` / `reverse`) в GUI намеренно не
выведены — они доступны в CLI (см. `examples/README.md`).

## Попробовать на готовом примере

В папке `examples/` уже лежат `employees.xlsx` и `order_template.docx`.
На вкладке **Генерация**:

1. **Таблица (.xlsx)** → выбрать `examples/employees.xlsx`.
2. **Шаблон (.docx)** → выбрать `examples/order_template.docx`.
3. **Папка результата** → любая папка для вывода (например `examples/output`).
4. Нажать **«Сгенерировать»**.

Имена файлов сформируются автоматически по колонкам ФИО
(`Иванов_Пётр_Семёнович.docx`), лист берётся активный. Нужны нестандартные
имя/лист/конфиг — это доступно в CLI (`examples/README.md`).

Внизу окна побежит прогресс-бар (`N/Total`), а по завершении появится
строка «Готово: 3 документ(ов) в …». На больших пакетах (тысячи строк)
окно не «замерзает» — генерация идёт в фоне, видно движение, можно
прервать кнопкой **«Отмена»** (уже созданные файлы останутся).

## Совсем с нуля

Нет своих файлов? Стартовый набор создаётся из CLI:

```bash
PYTHONPATH=src uv run python -m dyak init --dir ./my_project
```

Появятся `table.xlsx`, `template.docx`, `dyak.yaml`. Отредактируйте их под
себя (в шаблоне удалите блок «ШПАРГАЛКА») и укажите на вкладке
**Генерация**.

## Если что-то пошло не так

Ошибка показывается понятным текстом в нижней панели (без «непонятной
трассировки»); полный вывод программы там же можно прокрутить. Типичное:

- «Не распознаны колонки ФИО» — проверьте заголовки таблицы.

## Как это устроено

GUI ничего не считает сам: на каждое действие он запускает то же ядро
`dyak`, что и в терминале (`python -m dyak …`), и показывает его вывод.
Прогресс генерации берётся из машиночитаемого потока `--progress-json`
(см. раздел «Прогресс на больших объёмах» в `examples/README.md`).
Поэтому поведение GUI и CLI совпадает — окно это просто удобное «лицо».
