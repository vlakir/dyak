# Spec: T001 — Этап 0 «Скелет»

## 1. Overview

Минимальный сквозной пайплайн «таблица → документы»: прочитать xlsx,
по каждой строке отрендерить docx-шаблон простой строковой подстановкой
и записать файл с шаблонизируемым именем. **Склонения на этом этапе
нет** — это каркас, на который этапы T002–T005 навешивают
морфологию. Цель — рабочая команда `dyak generate` и проверенная
архитектура портов, чтобы движки склонения подключались без переделки
ядра.

## 2. User Stories

- Как кадровик, я хочу подать таблицу сотрудников и docx-шаблон и
  получить по документу на строку, чтобы не копировать данные руками.
- Как кадровик, я хочу задать маппинг «колонка таблицы → имя в шаблоне»
  в конфиге, чтобы шаблоны не зависели от точных заголовков таблицы.
- Как кадровик, я хочу управлять именем выходного файла шаблоном, чтобы
  документы складывались с понятными именами.

## 3. Functional Requirements

- ДОЛЖНА: команда `dyak generate --table <xlsx> --template <docx>
  --out <dir> [--config dyak.yaml] [--sheet <name>]`.
- ДОЛЖНА: читать xlsx через `openpyxl` (первая строка — заголовки),
  сопоставлять колонки по `config.columns`, лист — `--sheet` или
  активный.
- ДОЛЖНА: валидировать наличие обязательных колонок и непустоту
  обязательных ячеек ДО обработки; ошибка — с номером строки.
- ДОЛЖНА: рендерить шаблон через `docxtpl`, подставляя значения
  **строкой ровно как в ячейке** (без переформатирования, без склонения).
- ДОЛЖНА: рендерить имя выходного файла из `config.filename` тем же
  Jinja-движком.
- ДОЛЖНА: при коллизии имён добавлять суффикс `_2`, `_3`, … +
  предупреждение; создавать `--out` при отсутствии.
- ДОЛЖНА: прятать чтение данных за портом (источник) и рендер так, чтобы
  T002–T005 добавляли склонение без правки ядра.
- НЕ ДОЛЖНА (на этом этапе): склонять по падежам, знать про пол,
  petrovich/pymorphy3, overrides, команды `check`/`init`, PDF.

## 4. Success Criteria

- `dyak generate` на примере из `CONCEPT.md` §20 создаёт N файлов для
  N строк таблицы; подставленный текст совпадает с ячейками.
- Маппинг колонок и шаблон имени файла берутся из `dyak.yaml`
  (pydantic-валидация; неизвестные ключи → ошибка).
- Коллизия имён не теряет документы (суффикс + предупреждение).
- 4 проверки зелёные (`ruff check`, `ruff format --check`, `mypy`,
  `pytest`), coverage ≥ 80% на `src/`.

## 5. Key Entities

- **Config** — pydantic-модель: `columns` (маппинг), `filename`
  (шаблон Jinja), плюс задел под `gender_values`/`overrides` (валидация
  есть, поведение — позже). См. `CONCEPT.md` §9.
- **Record / Person (минимальный)** — строка таблицы как набор полей по
  русским именам-ключам; на этапе 0 все значения — строки. Структурные
  `Fio`/`Position`/`Gender` вводятся в T002–T003.
- **Excel Reader (порт)** — `xlsx → list[record]` с маппингом и
  валидацией.
- **Renderer** — обёртка `docxtpl` + Jinja-окружение (русские фильтры
  склонения добавляются в T002).
- **Writer** — имя файла из `config.filename` + запись docx + разрешение
  коллизий.

## 6. Assumptions & Constraints

- Платформа — Linux; чистый Python; Python 3.14 (`DECISIONS.md`).
- Источник данных — только xlsx (порт оставляем под CSV/БД, не
  реализуем).
- Детерминированность: один вход → побитово тот же набор документов.
- Структура — `src/dyak/` (НЕ `dyak/` в корне, как нарисовано в
  `CONCEPT.md` §16 — там «старая» раскладка; методика требует `src/`).

## 7. Out of Scope

- Склонение по падежам и всё, что с ним связано (T002–T004).
- Команды `check`, `init` (T004, T005).
- PDF-экспорт (T005).
- Согласование по полу, фильтр `согл` (T004).
- Редактирование/вёрстка шаблона, GUI, источники кроме xlsx.
