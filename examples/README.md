# Пример запуска dyak

Готовый набор для ручного прогона генерации документов.

| Файл | Назначение |
|---|---|
| `employees.xlsx` | Таблица сотрудников (3 строки). |
| `order_template.docx` | Шаблон приказа с русскими тегами `{{ … }}`. |
| `dyak.yaml` | Конфиг: маппинг колонок + шаблон имени файла. |

## Запуск

Из корня проекта:

```bash
uv sync   # один раз, если ещё не делали

PYTHONPATH=src uv run python -m dyak generate \
  --table examples/employees.xlsx \
  --template examples/order_template.docx \
  --out examples/output \
  --config examples/dyak.yaml
```

Результат — три файла в `examples/output/`:

```
Приказ_17_Иванов.docx
Приказ_18_Петрова.docx
Приказ_19_Ким.docx
```

Имя файла собирается из `filename` в `dyak.yaml`
(`Приказ_{{ номер_приказа }}_{{ сотрудник.фамилия }}.docx`).

## Что попробовать

- Поменять текст `order_template.docx` — добавить любые теги:
  `{{ сотрудник.фамилия }}`, `{{ сотрудник.имя }}`,
  `{{ сотрудник.отчество }}`, `{{ сотрудник.должность }}`, а также
  дополнительные колонки по их русскому имени из `dyak.yaml`
  (`{{ дата_начала }}`, `{{ номер_приказа }}` …).
- Добавить строки в `employees.xlsx` — на каждую получится отдельный
  документ.
- Изменить `filename` в `dyak.yaml` — поменяется схема имён выходных
  файлов.

## Важно (этап 0)

Сейчас реализована **простая подстановка без склонения**: значения
вставляются ровно как в таблице. Поэтому в шаблоне пока **нельзя**
использовать падежные фильтры (`{{ … | вн }}`, `| рд` и т.п.) и
`инициалы` — они появятся в T002 (этап «ФИО»). Тэг с неизвестным
фильтром приведёт к ошибке рендера.
