## 1. Overview

Дать кадровику контроль качества ДО генерации сотни файлов. Команда
`dyak check` делает сухой прогон (рендер в память, без записи) и печатает
отчёт: какие слова не просклонялись, где пол определён неуверенно или
расходится с ручным указанием, какие переменные шаблона не найдены. Плюс
два средства корректности рендера: фильтр согласования по полу
`{{ ФИО | согл('назначен','назначена') }}` и строгий режим Jinja
(`StrictUndefined`) — неизвестная переменная больше не подставляется молча
пустотой, а явно называется.

## 2. User Stories

- Как кадровик, перед генерацией приказов на 500 человек я хочу прогнать
  `dyak check` и увидеть строки, где склонение/пол сомнительны, чтобы
  поправить таблицу или `overrides`, а не печатать 500 битых файлов.
- Как кадровик, я хочу `{{ ФИО | согл('назначен','назначена') }}` →
  «назначена» для женщины, чтобы причастия/глаголы согласовались по полу.
- Как кадровик, опечатавшийся в теге (`{{ Должнсть }}`), я хочу понятную
  ошибку с именем переменной, а не молчаливый пропуск.

## 3. Functional Requirements

- ДОЛЖНА: команда `dyak check --table … --template … [--config …]
  [--sheet …]` — читает таблицу, строит контекст и **рендерит каждую
  строку в память** (без записи файлов), собирает и печатает отчёт.
- ДОЛЖНА: отчёт включает по строкам:
  - **неизвестные переменные шаблона** (через `StrictUndefined`) — с
    именем переменной и номером строки;
  - **не просклонившиеся склоняемые значения** — `Declinable`, который ни
    в одном косвенном падеже не меняется (кандидат на `override` либо
    легитимно несклоняемый — кадровик решает);
  - **пол определён неуверенно** (сработал дефолт) или **расходится** с
    ручным `genders` (ручной ≠ авто по отчеству/имени).
- ДОЛЖНА: регистрировать фильтр `согл(m, f)` — по полу `Declinable`-объекта
  (ФИО) выбирает мужскую/женскую форму.
- ДОЛЖНА: Jinja-окружение в режиме `StrictUndefined`; неизвестная
  переменная при `generate`/`check`/имени файла → явная ошибка
  `TemplateError` с именем (а не пустая подстановка).
- ДОЛЖНА: `check` завершается кодом `0`, если фатальных проблем нет
  (отчёт — предупреждения), и кодом `1` при фатальных (неизвестная
  переменная, ошибка чтения таблицы) — чтобы годилось для CI/скриптов.
- НЕ ДОЛЖНА: чинить найденное автоматически (только сообщает); менять
  формат вывода `generate`.

## 4. Success Criteria

- `dyak check` на корректной таблице/шаблоне печатает «проблем не
  найдено» и выходит с кодом 0; файлов не создаёт.
- Несклоняемая/непросклонившаяся должность (напр. аббревиатура) попадает
  в отчёт с номером строки и значением.
- Неоднозначный пол («Саша» без отчества) → строка в отчёте «пол не
  определён, взят мужской».
- Ручной `genders` пол, противоречащий отчеству, → строка «ручной пол ≠
  определённый по отчеству».
- `{{ ФИО | согл('назначен','назначена') }}` даёт «назначен»/«назначена»
  по полу.
- Опечатка в теге → `check` (и `generate`) падает понятной ошибкой с
  именем переменной, exit code 1.
- 4 проверки зелёные, coverage ≥ 80%. `examples/` показывает `согл`.

## 5. Key Entities

- **CheckReport** — собранные находки: список `Issue { row, kind,
  message }` (kinds: `undefined` / `not_declined` / `gender_ambiguous` /
  `gender_mismatch`). Печатается человекочитаемо; пустой → «ОК».
- **GenderResolution** — результат определения пола: `gender` + `source`
  (`override`/`patronymic`/`name`/`default`). `detect_gender` остаётся
  тонкой обёрткой над ним для горячего пути.
- **согл-filter** — `согл(value, male, female)`: по полу `value` (ФИО)
  возвращает нужную форму.
- **StrictUndefined env** — общий `build_jinja_env` переключается в
  строгий режим; рендер ловит `UndefinedError` и заворачивает в доменную
  `TemplateError` с именем переменной.

## 6. Assumptions & Constraints

- «Не просклонилось» — эвристика (форма == исходной во всех косвенных).
  Возможны ложные срабатывания на легитимно несклоняемых (Дюма, Ким-ж,
  -ко) — отчёт advisory, кадровик решает. Это сознательно (CONCEPT §13).
- `согл` опирается на пол `Declinable` (ФИО). В строке без ФИО `согл`
  неприменим → понятная ошибка (редкий кейс для кадровых шаблонов).
- `check` рендерит в память тем же путём, что `generate` (один источник
  истины), отличие — нет записи на диск и есть сбор отчёта.

## 7. Out of Scope

- Автоисправление найденных проблем.
- PDF/`init`/прогресс/оптимизация — T005.
- Полный синтаксический анализ согласования прилагательных — закрывается
  `overrides` (R1), не `check`.
- Машиночитаемый формат отчёта (`--json`) — относится к контракту CLI↔GUI
  (T005/T008); в T004 отчёт человекочитаемый.

## Clarify (ответы — решения ВрИО Гвидо, ночной прогон 2026-06-22)

### Resolved

- **Q: Где `согл` берёт пол в плоском контексте (нет объекта
  `сотрудник` из CONCEPT §8.3)?**
  A: с `Declinable`-ФИО: `{{ ФИО | согл('м','ж') }}`. `Fio` уже знает пол
  (через `NamePart.gender`) — добавляю `Fio.gender`. Это ближайший к
  CONCEPT плоский аналог; объект-«сотрудник» T006 упразднил.

- **Q: Что считать «пол-колонка ≠ пол-по-отчеству», раз колонки пола
  нет (T006)?**
  A: сверяем **ручной `genders`** с **авто-определением** (отчество/имя).
  Если ручной задан и расходится с уверенным авто — в отчёт. Плюс
  отдельно — случаи, где авто сработало дефолтом (неуверенно).

- **Q: `StrictUndefined` ломает `generate`?**
  A: это и есть цель «надёжности» — молчаливые пустые подстановки
  опасны. Перевожу общий env в строгий режим; рендер ловит
  `UndefinedError` → доменная `TemplateError` с именем переменной (и
  generate, и check, и имя файла).

- **Q: «Не просклонилось» — как детектить без ложного шума?**
  A: флагируем `Declinable`, который **ни в одном** косвенном падеже не
  меняется (полностью несклоняемый движком). Частичные огрехи согласования
  не ловим автоматически (R1 → overrides). Отчёт advisory.

- **Q: Коды возврата `check`?**
  A: `0` — фатальных нет (только warnings-отчёт); `1` — фатальное
  (неизвестная переменная, ошибка таблицы). Годится для скриптов/CI.

## Analyze (Гвидо)

- 🟡 **W1.** `StrictUndefined` глобально может «уронить» шаблоны, где
  кадровик намеренно опускает редкое поле. Митигируем: ошибка понятная и
  указывает строку+имя; `check` показывает все такие места разом до
  `generate`. Это осознанный trade-off «надёжности».
- 🟢 **N1.** Рендер в память: `DyakTemplate.render` уже строит docx
  в памяти; для check достаточно не вызывать `.save`. Вынесу общий
  `render_to_document(template, context) -> Document`, а `render_document`
  = он + save. DRY, один путь рендера.
- 🟢 **N2.** Перебор `Declinable` для «не склонилось»: после
  `build_context` итерируем значения контекста, `isinstance(v, Declinable)`
  (порт `runtime_checkable`), пробуем 5 косвенных падежей.
- 🟢 **N3.** `Fio.__getattr__` уже перехватывает неизвестные атрибуты —
  добавление обычного свойства `gender` безопасно (его вернёт обычный
  доступ, до `__getattr__`). Проверить, что `gender` не конфликтует с
  `__getattr__` (не конфликтует: реальные атрибуты приоритетнее).
- 🟢 **N4.** `resolve_gender` (с `source`) — рефактор `gender.py`;
  `detect_gender` тонкая обёртка. Существующие тесты T002 не ломаются.
- 🟢 **N5.** `согл` и падежные фильтры — в одном `build_jinja_env`.
  `согл(value, m, f)`: `value.gender` → форма; нет `gender` → понятная
  ошибка.

Проход чистый — реализуем по фазам.
