## Модульная раскладка (`src/dyak/reverse/`)

Новый подпакет — reverse это самостоятельная подсистема, зеркальная forward.

- `reverse/__init__.py` — публичный API (`build_template`, `ReverseReport`).
- `reverse/candidates.py` — из `Person`+контекста собрать **кандидатов на
  замену**: для каждого значения (роль/ключ) набор искомых форм →
  `(тег, набор_форм, склоняемый?)`. Переиспользует `build_context`
  (`Fio`/`NamePart`/`Position` + плоские `cells`) и `resolve_row_gender`.
  Для склоняемых — `inflect(case)` по 6 падежам (decline-and-match);
  омонимия форм фиксируется здесь (несколько падежей → одна строка).
- `reverse/matcher.py` — поиск кандидатов в тексте параграфа и **разрешение
  перекрытий**: собрать все спаны, длинные раньше, жадно занять
  непересекающиеся (Analyze W4); короткие/числовые — словная граница, при
  множественности → `ambiguous`, замена только при единственном совпадении
  (Q2).
- `reverse/docx_rewrite.py` — **run-split переписывание** (см. ниже) +
  подсветка (`w:highlight`) спорных run'ов (Q4).
- `reverse/report.py` — `ReverseReport`/`Finding`/категории по образцу
  `check.py` (`replaced`/`ambiguous`/`not_found`/`unmatched_text`/
  `roundtrip_mismatch`), `format_report`.
- `reverse/engine.py` — оркестратор: читает docx, по параграфам матчит и
  переписывает, round-trip verify, возвращает (`Document`, `ReverseReport`).

## Run-split переписывание (ядро сложности)

Проблема: Word дробит текст по `run`'ам произвольно; искомая форма
(«Иванову») может быть разрезана на `[Ива][нову]` в двух run'ах, а целевой
тег `{{ Фамилия | дт }}` надо вставить целиком и валидно.

Алгоритм на параграф (инверсия склейки docxtpl):
1. Склеить текст всех run'ов параграфа → плоская строка + карта
   `offset → (run_index, offset_внутри_run)`.
2. На плоской строке найти непересекающиеся спаны-замены (matcher).
3. Для каждого спана (с конца, чтобы офсеты не съезжали): определить
   затронутые run'ы; в **первый** run спана положить `текст_до + ТЕГ`,
   из последнего — `текст_после`, промежуточные run'ы спана очистить
   (`run.text = ''`). Так тег целиком оказывается в одном run → docxtpl при
   forward-рендере увидит его неразрезанным.
4. Подсветка: если спан `ambiguous`/`unmatched` — выставить
   `run.font.highlight_color` на run с тегом/фрагментом.

Обходим **все** параграфы, включая вложенные таблицы (готовый
`_iter_paragraphs` из `render/engine.py` — переиспользовать/вынести).

## Round-trip verify

После сборки шаблона (in-memory `Document`): сохранить во временный буфер →
`generate`-рендер на той же строке (`render_to_document`) → сравнить
извлечённый текст параграфов (нормализованные пробелы, Q5) с исходным
документом. Несовпадения → `roundtrip_mismatch` в отчёт. Полнота не
гарантируется (Analyze W2) — это про верность, не про покрытие.

## CLI

`dyak reverse --doc <docx> --table <xlsx> --row N --out <docx>`
`[--sheet] [--config]` (N2). Печатает `format_report`, код 1 при фатальных
(не открылся docx/таблица/строки нет), иначе 0 даже при warning'ах
(best-effort — шаблон всё равно сохранён).

## Фазы (коммиты на ветке)

1. **Каркас + точные совпадения.** Подпакет, CLI `reverse`, чтение docx,
   run-split переписывание, матчинг **только точных строковых значений**
   (плоские ячейки, именительные формы) → `{{ тег }}`, отчёт
   `replaced`/`not_found`/`unmatched_text`, базовые тесты (вкл. run-split).
2. **Decline-and-match.** Склоняемые ФИО/должности по 6 падежам → тег с
   падежным фильтром; ФИО по частям (Q3); омонимия → первый падеж +
   `ambiguous` + подсветка (Q4); тесты на падежи (регрессионная фикстура).
3. **Round-trip + полировка.** Round-trip verify + `roundtrip_mismatch`;
   пример в `examples/` (реально прогнать); `CHANGELOG.md` (`[Unreleased]`);
   ADR в `DECISIONS.md` (подсистема reverse, run-rewriting, отказ от slovnet
   в пользу decline-and-match). 4 проверки зелёные, coverage ≥ 80%.
