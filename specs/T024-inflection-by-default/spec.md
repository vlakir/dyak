# Spec: Склонение по умолчанию (универсальный фраз-движок)

**Статус:** Analyzed
**Дата создания:** 2026-06-25
**Связанные документы:** ADR «T024 …» в `DECISIONS.md`; суперсёдит T020,
поглощает T022; риск-зона — reverse (T007).

---

## 1. Overview

Сейчас склонение — «привилегия по роли»: значение склоняется, только если
колонка распознана как ФИО / должность / звание (`filters.py` зовёт
`.inflect` лишь на `Declinable`-объектах, которые навешиваются по ролям).
Любая новая колонка (например «Подразделение») проходит несклонённой, а
конкуренция за роль рождает баги (T020: «Подразделение» крадёт роль у
«Должности»). Фича переворачивает дефолт: **склоняется ЛЮБАЯ текстовая
колонка** универсальным фраз-движком, а роли усыхают до специализаций
(ФИО — petrovich+пол; звание — спец-движок; коды/даты — литерал).

## 2. Сценарии использования

- Кадровик пишет в шаблоне `{{ Подразделение | вн }}` для колонки, которой
  раньше не было «роли», и получает корректный винительный — без правки
  конфига и без объявления роли.
- Многословные должности склоняются грамматически: «рота связи» → «роту
  связи» (не «роту связь»), «1 мотострелковый батальон» в винительном
  остаётся собой (неодушевлённость), составные «механик-водитель» →
  «механику-водителю».
- Любая «оригинальная» текстовая колонка (примечание, основание) при
  применении падежного фильтра склоняется best-effort.

## 3. Functional Requirements

- ДОЛЖНА: склонять любую текстовую ячейку при падежном фильтре, без
  объявления роли (модель «склонение по умолчанию»).
- ДОЛЖНА: для многословной фразы склонять голову и согласованные
  модификаторы (прилагательные/числительные), **замораживая** несогласованный
  родительный хвост («рота **связи**», «начальник **штаба**»).
- ДОЛЖНА: учитывать **одушевлённость** в винительном (неодуш. муж./ср.:
  вн = им. — «танковый батальон» не меняется).
- ДОЛЖНА: корректно склонять **составные должности** с любыми разделителями
  (см. §3a).
- ДОЛЖНА: сохранять ФИО на petrovich+пол (склонение по полу), звание — на
  `RankInflector` (голова + замороженный хвост, дефис — последний сегмент).
- ДОЛЖНА: НЕ склонять литералы — коды/личные номера/даты (по форме значения).
- МОЖЕТ: принимать ручной `overrides.position` (по тексту фразы) с приоритетом
  над движком — для несогласуемых редких фраз.
- НЕ ДОЛЖНА: ломать reverse (T007) — все его тесты зелёные.
- НЕ ДОЛЖНА: регрессировать ФИО / звания / существующие должности.

### 3a. Составные должности (разные разделители)

Кадровик пишет разделители как попало; распознаём все виды чёрточек
(`-` U+002D, `‐`/`‑`, `–` U+2013, `—` U+2014, `―`, `−` U+2212 и пр.).

- **Разделитель-с-пробелами** (` - `, ` – `, ` — `) → ДВЕ независимые
  параллельные должности: каждый сегмент склоняется как отдельная фраза,
  собирается обратно С ТЕМ ЖЕ разделителем (символ-в-символ, без нормализации).
- **Тесный дефис** без пробелов → неоднозначно:
  - известная pymorphy лексема («генерал-майор», «контр-адмирал»-как-звание)
    → склоняем целиком (меняется хвост): «генерал-майору»;
  - неизвестная → аппозитивный компаунд, склоняем КАЖДУЮ часть:
    «стрелок-радист» → «стрелку-радисту».
- Пробел ВНУТРИ сегмента без чёрточки — обычная фраза (голова+согласование+
  замороженный родительный хвост), НЕ разделитель.

## 4. Success Criteria (эталон приказа + составные)

Пример приказа (`примеры для дьяка.zip`; шаблон
`{{ Должность|дт }} {{ Подразделение|вн }} {{ Звание|дт }} {{ Фамилия|дт }}`):

- «командиру 1 мотострелковый батальон подполковнику ПУПКИНУ»
- «телефонисту роту связи ефрейтору БИДЕ» (БИДЕ несклоняемая)
- «стрелку взвод охраны рядовому СЕМЕНОВУ»
- «механику-водителю танковый батальон младшему сержанту БИВЕНЮ»

Юниты фраз-движка (|дт, если не указано):

- «стрелок-радист» → «стрелку-радисту»
- «телефонист - линейный надсмотрщик» → «телефонисту - линейному надсмотрщику»
- «начальник штаба — заместитель командира» → «начальнику штаба —
  заместителю командира» (em-dash сохранён; штаба/командира заморожены)
- «генерал-майор» → «генерал-майору» (только хвост — известная лексема)
- разные разделители (-, –, —, с пробелами и без) — символ-в-символ.

## 5. Key Entities

- `PhraseInflector` — универсальный движок (морфоразбор pymorphy3).
- `Phrase` — `Declinable`-обёртка над любой текстовой колонкой (+ overrides).
  Заменяет `Position` (тот был «роль-привратник», теперь — любая фраза).
- `NamePart`/`Fio` (petrovich+пол), `Rank`/`RankInflector` — без изменений.

## 6. Assumptions & Constraints

- pymorphy3 на Python 3.14 — рабочий (используется с T003).
- Идеальный синтаксический разбор не требуется: целимся в типичные кадровые
  фразы (голова + родительный комплемент, согласованные определения).
- «Литерал» определяется по ФОРМЕ значения (регэкспы кода/даты), не по роли:
  это снимает T020-аналог для двух код-колонок (№ п/п и л/н) без правки
  `columns.py`.

## 7. Out of Scope

- Полноценный синтаксический парсер; редкие/неоднозначные фразы закрываются
  ручным `overrides.position`.
- Нейро-морфология (slovnet, T015 — отложено).
- Изменение пользовательского ключа конфига `overrides.position`.
- Объединение `RankInflector` в общий движок (звание остаётся спец-веткой,
  чтобы не регрессировать «контр-адмирал»-компаунды; ADR).

## Clarify (ответы — разумные дефолты автономного прогона, на ревью утром)

### Q1. Голова и замороженный хвост — морфоанализ или эвристика?

**Решение:** морфоанализ pymorphy3. Хвост = максимальный правый прогон слов,
у которых верхний разбор родительный И нет именительного ЕДИНСТВЕННОГО
разбора. Это различает настоящий комплемент («связи», «охраны» — морозим) и
голову-омоним с родительным множественного («батальон», «техник» — склоняем).
Подтверждено на 28 кейсах прототипа.

### Q2. Одушевлённость для винительного?

**Решение:** pymorphy склоняет существительное-голову с её родной
одушевлённостью; согласованным модификаторам передаём грамему одушевлённости
головы (`inflect({case, anim})`), поэтому «танковый батальон»|вн = им.

### Q3. Литерал — по роли или по форме значения?

**Решение:** по форме значения (регэкспы кода/даты в `context.py`). Не трогаем
`columns.py`/`Recognition` — меньше риск, и покрываются ОБА код-столбца
(победивший роль и проигравший её), что снимает T020-аналог для личных
номеров.

### Q4. Взаимодействие с reverse (T007)?

**Решение:** reverse переиспользует `build_context`, поэтому generic-обёртка
автоматически даёт reverse больше склоняемых кандидатов (любая текстовая
колонка). Держим reverse рабочим: все тесты зелёные; если generic-обёртка
конфликтует глубоко — адаптируем консервативно или останавливаемся на узле
(см. риск в плане). Литералы (коды/даты) не оборачиваются → не зашумляют
reverse ложными падежными кандидатами.

### Q5. Звание — обобщать в общий движок?

**Решение:** НЕТ. `RankInflector` остаётся спец-веткой: общий движок
«известная лексема → целиком / иначе → каждая часть» регрессировал бы
«контр-адмирал» (нужен только последний сегмент). Цена — небольшое дублирование
логики заморозки хвоста; выгода — нулевой риск регресса званий.
